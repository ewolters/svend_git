## **1. SCOPE AND PURPOSE**

### **1.1 Purpose**

DAT-001 defines the canonical data model architecture for Kjerne/Svend. All domain entities, base classes, ownership patterns, field conventions, indexing strategies, and migration rules are codified here. New models MUST conform to these patterns; deviations require documented exceptions.

**Core Principle:**

> Data models are the skeleton of the system. Every field, every index, every constraint is a decision with downstream consequences.
> If a model doesn't trace back to this standard, it's technical debt.

### **1.2 Scope**

**Applies to:**
- All Django models in `core/` (target data model)
- Abstract base classes in `syn/core/base_models.py`
- SDK models built on `SynaraEntity`, `SynaraRegistry`, `SynaraImmutableLog`
- Migration strategy from `agents_api.Problem` to `core.Project`

**Does NOT apply to:**
- Legacy models in `agents_api/models.py` (governed by migration plan, not new development)
- Third-party Django models (auth, sessions, admin)
- Forge, Tempora, or other service-specific models that predate this standard (will be migrated per roadmap)

### **1.3 Implementation**

|-----------|----------|---------|
| `SynaraEntity` | `syn/core/base_models.py` | Abstract base for all domain entities |

## **3. TERMINOLOGY**

<!-- table: terminology -->

| Term | Definition |
|------|------------|
| **SynaraEntity** | Abstract base class providing UUID PK, correlation tracking, tenant isolation, audit timestamps, soft delete, event emission, and metadata. All new domain models inherit from it. |
| **SynaraRegistry** | Abstract base class for database-backed enumeration values with code/label, active flag, display order, and tenant scoping. |
| **SynaraImmutableLog** | Abstract base class for write-once audit logs with hash chain integrity and tamper detection. |
| **Soft delete** | Logical deletion that sets `is_deleted=True` and preserves the record for audit purposes. Physical deletion requires explicit `hard_delete=True`. |
| **User XOR Tenant** | Ownership constraint requiring exactly one of `user` or `tenant` to be set on a resource, never both, never neither. Enforced by `CheckConstraint`. |
| **Soft FK** | A cross-app reference using `target_type` CharField + `target_id` UUIDField instead of a database foreign key, used when FK would create circular dependencies. |
| **Likelihood ratio** | P(evidence | H true) / P(evidence | H false). Values > 1 support the hypothesis; values < 1 oppose it. Central to Bayesian updating in the evidence model. |
| **Dual-write** | Migration strategy where writes go to both legacy (`agents_api.Problem`) and target (`core.Project`) systems simultaneously. |

## **4. PRIMARY KEYS & IDENTITY**

### **4.1 UUID v4 Strategy**

<!-- assert: All new domain entities use UUID v4 primary keys | check=dat-uuid-pk -->
<!-- impl: syn/core/base_models.py:SynaraEntity -->
<!-- impl: core/models/project.py:Project -->
<!-- impl: core/models/hypothesis.py:Hypothesis -->
<!-- impl: core/models/hypothesis.py:Evidence -->
<!-- impl: core/models/hypothesis.py:EvidenceLink -->
<!-- impl: core/models/tenant.py:Tenant -->
<!-- impl: core/models/tenant.py:Membership -->
<!-- impl: core/models/graph.py:KnowledgeGraph -->
<!-- impl: core/models/graph.py:Entity -->
<!-- impl: core/models/graph.py:Relationship -->
<!-- rule: mandatory -->

All new domain entities MUST use UUID v4 primary keys:

<!-- code: correct -->
```python
id = models.UUIDField(
    primary_key=True,
    default=uuid.uuid4,
    editable=False,
)
```

**Rationale:**
- No enumeration attacks (sequential IDs leak record count)
- Safe for distributed generation (no central sequence)
- URL-safe for API resources
- Consistent with tenant_id and correlation_id (all UUIDs)

### **4.2 Documented Exceptions**

<!-- assert: SysLogEntry uses BigAutoField for sequential ordering | check=dat-syslog-bigauto -->
<!-- impl: syn/audit/models.py:SysLogEntry -->

| Model | PK Type | Reason |
|-------|---------|--------|
| `SysLogEntry` | `BigAutoField` | Sequential ordering matters for hash chain integrity (AUD-001 §4.1) |
| `agents_api.DSWResult` | `CharField(max_length=50)` | Legacy model, will be deprecated |
| `agents_api.TriageResult` | `CharField(max_length=50)` | Legacy model, will be deprecated |

New models MUST NOT use `AutoField`, `BigAutoField`, or `CharField` as primary keys unless a documented exception is approved and recorded here.

### **4.3 Correlation ID**

<!-- assert: SynaraEntity provides unique correlation_id for causal tracing | check=dat-correlation-id -->
<!-- impl: syn/core/base_models.py:SynaraEntity -->

Every `SynaraEntity` carries a `correlation_id` (UUID v4, unique, indexed) for distributed tracing and causal chain tracking. This is separate from the primary key — the PK identifies the record; the correlation_id identifies it in the causal graph.

<!-- code: correct -->
```python
correlation_id = models.UUIDField(
    default=uuid.uuid4,
    unique=True,
    db_index=True,
    editable=False,
)
```

### **5.4 SynaraMeta Configuration**

<!-- assert: SynaraEntity subclasses configure behavior via SynaraMeta inner class | check=dat-synara-meta -->

<!-- code: correct -->

class Claim(SynaraEntity):

| `terminal_states` | list[str] | States that cannot transition further |
| `require_governance` | bool | Whether state changes require governance approval |

## **6. OWNERSHIP & TENANCY**

### **6.1 User XOR Tenant Constraint**

<!-- assert: Project and KnowledgeGraph enforce User XOR Tenant ownership via CheckConstraint | check=dat-user-xor-tenant -->
<!-- impl: core/models/project.py:Project -->
<!-- impl: core/models/graph.py:KnowledgeGraph -->
<!-- rule: mandatory -->

Resources with dual ownership (individual or team) MUST enforce exactly-one-owner via `CheckConstraint`:

<!-- code: correct -->
```python
class Meta:
    constraints = [
        models.CheckConstraint(
            check=(
                models.Q(user__isnull=False, tenant__isnull=True) |
                models.Q(user__isnull=True, tenant__isnull=False)
            ),
            name="project_has_single_owner",
        )
    ]
```

**Current models with this constraint:**

| Model | Constraint Name | Fields |
|-------|----------------|--------|
| `Project` | `project_has_single_owner` | `user` XOR `tenant` |
| `KnowledgeGraph` | `graph_has_single_owner` | `user` XOR `tenant` |

<!-- code: prohibited -->
```python
# PROHIBITED: No ownership constraint
class Project(models.Model):
    user = models.ForeignKey(User, null=True, blank=True, ...)
    tenant = models.ForeignKey(Tenant, null=True, blank=True, ...)
    # Missing CheckConstraint — allows both null or both set
```

## **7. FIELD PATTERNS**

### **7.1 Timestamp Conventions**

<!-- assert: Timestamps use _at suffix, auto_now_add for creation, auto_now for modification | check=dat-timestamp-pattern -->
<!-- impl: syn/core/base_models.py:SynaraEntity -->
<!-- impl: core/models/project.py:Project -->
<!-- rule: mandatory -->

| Pattern | Field | Django Config |
|---------|-------|---------------|
| Creation time | `created_at` | `DateTimeField(auto_now_add=True, db_index=True)` |
| Modification time | `updated_at` | `DateTimeField(auto_now=True)` |
| Domain event time | `resolved_at`, `completed_at`, `joined_at` | `DateTimeField(null=True, blank=True)` |

**Rules:**
- Timestamp fields MUST use `_at` suffix (not `_date`, `_time`, `_timestamp`)
- `created_at` MUST be indexed (`db_index=True`) for recency sort
- `updated_at` SHOULD NOT be indexed unless queried directly (it changes on every save)

### **7.2 Actor / Identity Fields**

<!-- assert: created_by and updated_by use CharField(255) to avoid FK dependency | check=dat-actor-charfield -->
<!-- impl: syn/core/base_models.py:SynaraEntity -->
<!-- rule: mandatory -->

Actor fields (`created_by`, `updated_by`, `deleted_by`) MUST use `CharField(max_length=255)` — NOT a ForeignKey to the User model.

**Rationale:**
- Avoids circular FK dependency between `syn/` SDK layer and Django auth
- Actors may be system processes (e.g., `"system"`, `"migration"`)
- String identity is sufficient for audit purposes

<!-- code: correct -->
```python
created_by = models.CharField(max_length=255, blank=True, default="")
updated_by = models.CharField(max_length=255, blank=True, default="")
```

<!-- code: prohibited -->
```python
# PROHIBITED: FK to User creates circular dependency in SDK base
created_by = models.ForeignKey(settings.AUTH_USER_MODEL, ...)
```

**Exception:** Domain models in `core/` MAY use FK to `AUTH_USER_MODEL` for `created_by` when the relationship is a direct domain concern (e.g., `Hypothesis.created_by` FK, `Evidence.created_by` FK). The SDK base class fields remain CharFields.

### **7.3 JSONField Patterns**

<!-- assert: JSONField uses typed defaults (list or dict) and consistent structural patterns | check=dat-jsonfield-patterns -->
<!-- impl: core/models/project.py:Project -->
<!-- impl: core/models/hypothesis.py:Hypothesis -->

JSONFields serve specific structural roles. Each pattern has a canonical shape:

| Pattern | Default | Structure | Example Fields |
|---------|---------|-----------|----------------|
| List of strings | `default=list` | `["item1", "item2"]` | `problem_whats`, `problem_wheres`, `scope_in`, `tags` |
| List of objects | `default=list` | `[{name, role, department}]` | `team_members`, `milestones`, `resolution_actions` |
| Append-only changelog | `default=list` | `[{ts, action, detail, user}]` | `changelog`, `phase_history` |
| Probabilistic history | `default=list` | `[{probability, previous, evidence_id, likelihood_ratio, timestamp}]` | `probability_history` |
| State snapshot | `default=dict` | `{key: value}` | `before_snapshot`, `after_snapshot`, `changes` |
| Configuration bag | `default=dict` | `{key: value}` | `design_spec`, `metadata`, `settings`, `raw_output` |

**Rules:**

<!-- rule: mandatory -->

- JSONField MUST specify `default=list` or `default=dict` — never `default=None` for new fields
- JSONField containing lists MUST use `default=list`, not `default=[]` (mutable default protection)
- JSONField containing dicts MUST use `default=dict`, not `default={}`
- Nullable JSONField (`null=True`) is allowed only for optional state that has a meaningful "not set" vs "empty" distinction (e.g., `synara_state`, `interview_state`)

<!-- code: correct -->
```python
tags = models.JSONField(default=list, blank=True)
metadata = models.JSONField(default=dict, blank=True)
```

<!-- code: prohibited -->
```python
# PROHIBITED: mutable default
tags = models.JSONField(default=[], blank=True)
# PROHIBITED: None where empty is meaningful
tags = models.JSONField(default=None, null=True, blank=True)
```

### **7.4 Foreign Key Patterns**

<!-- assert: FK relationships follow hierarchical, M2M junction, self-referential, one-to-one, and soft FK patterns | check=dat-fk-patterns -->
<!-- impl: core/models/project.py:Project -->
<!-- impl: core/models/hypothesis.py:EvidenceLink -->
<!-- impl: core/models/graph.py:KnowledgeGraph -->

**Pattern catalog:**

| Pattern | Example | on_delete | Purpose |
|---------|---------|-----------|---------|
| Hierarchical | `Hypothesis.project -> Project` | CASCADE | Child deleted when parent deleted |
| M2M with metadata | `EvidenceLink` (hypothesis + evidence) | CASCADE | Junction table with likelihood_ratio, direction |
| Self-referential | `Hypothesis.merged_into -> self` | SET_NULL | Merge tracking |
| One-to-One | `KnowledgeGraph.user -> User` | CASCADE | Exactly one graph per user |
| Soft FK | `StudyAction.target_type + target_id` | N/A | Cross-app reference without DB constraint |
| Optional parent | `Evidence.related_entity -> Entity` | SET_NULL | Optional enrichment link |

**FK naming rules:**

<!-- rule: mandatory -->

- FK field names: use the natural name, NOT suffixed with `_fk` or `_id` (Django adds `_id` automatically)
- `related_name`: plural natural English (`hypotheses`, `evidence_links`, `memberships`)
- `on_delete` MUST be explicit on every FK:
  - `CASCADE`: child meaningless without parent
  - `SET_NULL`: optional reference, orphan is valid
  - `PROTECT`: prevent deletion if children exist (rare, use deliberately)

<!-- code: correct -->
```python
project = models.ForeignKey("core.Project", on_delete=models.CASCADE, related_name="hypotheses")
```

<!-- code: prohibited -->
```python
# PROHIBITED: _fk suffix
project_fk = models.ForeignKey("core.Project", ...)
# PROHIBITED: missing related_name
project = models.ForeignKey("core.Project", on_delete=models.CASCADE)
```

| Correct | Prohibited |

### **7.6 Naming Conventions**

<!-- rule: mandatory -->

| Element | Convention | Example |
|---------|-----------|---------|
| Table name | Explicit `db_table`, snake_case, app prefix | `core_project`, `core_hypothesis`, `core_evidence_link` |

## **8. INDEX & CONSTRAINT PATTERNS**

### **8.1 Index Strategy**

<!-- assert: Indexes follow ownership+filter, recency, correlation, and ranking patterns | check=dat-index-patterns -->
<!-- impl: core/models/project.py:Project -->
<!-- impl: core/models/hypothesis.py:Hypothesis -->
<!-- impl: core/models/hypothesis.py:Evidence -->
<!-- impl: core/models/graph.py:Entity -->
<!-- impl: core/models/graph.py:Relationship -->

**Index pattern catalog:**

| Pattern | Fields | Purpose | Models |
|---------|--------|---------|--------|
| Ownership + filter | `(user, status)`, `(tenant, status)` | List resources for an owner filtered by state | Project |
| Recency sort | `(-updated_at)`, `(-created_at)` | Default ordering queries | Project, Evidence |
| Correlation tracking | `(correlation_id)` | Distributed tracing lookups | SynaraEntity (base) |

| Status filter | `(project, status)` | Filter hypotheses by status within project | Hypothesis |
| Graph composite | `(graph, entity_type)`, `(graph, name)` | Entity lookups within a graph | Entity |
| Relationship lookup | `(graph, relation_type)`, `(source, relation_type)`, `(target, relation_type)` | Edge traversal | Relationship |

**Rules:**
- Every FK used in `filter()` queries SHOULD have an index (Django adds this by default for FK)
- Composite indexes: put the higher-cardinality field first for equality, lower-cardinality for range
- Do NOT index fields that change on every save (e.g., `updated_at`) unless specifically queried in WHERE clauses

### **8.2 Constraint Patterns**

<!-- assert: CheckConstraints enforce domain invariants at the database level | check=dat-check-constraints -->
<!-- impl: core/models/project.py:Project -->
<!-- impl: core/models/graph.py:KnowledgeGraph -->

| Constraint Type | Example | Purpose |
|----------------|---------|---------|
| `CheckConstraint` | `project_has_single_owner` | User XOR Tenant ownership |
| `CheckConstraint` | `graph_has_single_owner` | User XOR Tenant ownership |
| `unique_together` | `[["hypothesis", "evidence"]]` on EvidenceLink | No duplicate hypothesis-evidence pairs |
| `unique_together` | `[["tenant", "user"]]` on Membership | One membership per user per tenant |

**Rules:**

<!-- rule: mandatory -->

- Domain invariants that can be expressed as SQL constraints MUST be enforced at the database level, not only in application code
- Constraint names MUST be descriptive: `{model}_{describes_rule}` (e.g., `project_has_single_owner`)
- `unique_together` MUST be used for composite uniqueness (not application-level checks alone)

## **11. ANTI-PATTERNS**

### **11.1 AutoField Primary Keys on Domain Models**

<!-- rule: mandatory -->

**PROHIBITED:**
<!-- code: prohibited -->
```python
class Investigation(models.Model):
    id = models.AutoField(primary_key=True)  # Sequential, enumerable
```

**CORRECT:**
<!-- code: correct -->
```python
class Investigation(models.Model):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
```

Sequential IDs leak record counts, enable enumeration attacks, and conflict in distributed systems. Use UUID v4.

### **11.2 Missing Ownership Constraint**

<!-- rule: mandatory -->

**PROHIBITED:** Models with both `user` and `tenant` FKs but no `CheckConstraint` ensuring exactly one is set.

This allows records owned by both a user and a tenant, or owned by neither. Both states are invalid.

### **11.3 Mutable Default on JSONField**

<!-- rule: mandatory -->

**PROHIBITED:**
<!-- code: prohibited -->
```python
data = models.JSONField(default=[])   # Mutable default — shared across instances
data = models.JSONField(default={})   # Same problem
```

**CORRECT:**
<!-- code: correct -->
```python
data = models.JSONField(default=list)  # Callable — new list per instance
data = models.JSONField(default=dict)  # Callable — new dict per instance
```

### **11.4 FK Without related_name**

<!-- rule: mandatory -->

**PROHIBITED:**
<!-- code: prohibited -->
```python
project = models.ForeignKey("core.Project", on_delete=models.CASCADE)
# Generates project_set, which is ambiguous when multiple FKs to same model exist
```

**CORRECT:**
<!-- code: correct -->
```python
project = models.ForeignKey("core.Project", on_delete=models.CASCADE, related_name="hypotheses")
```

### **11.5 Application-Only Uniqueness**

<!-- rule: mandatory -->

**PROHIBITED:** Enforcing uniqueness only in Python (view-level checks) without a database constraint.

```python
# PROHIBITED: Race condition — two requests can create duplicates
if EvidenceLink.objects.filter(hypothesis=h, evidence=e).exists():
    return error("duplicate")
EvidenceLink.objects.create(hypothesis=h, evidence=e, ...)
```

**CORRECT:** Use `unique_together` or `UniqueConstraint` at the model level.

### **11.6 Hard Delete Without Audit Trail**

<!-- rule: mandatory -->

**PROHIBITED:** Calling `Model.objects.filter(...).delete()` on domain data without soft-delete or audit logging.

**CORRECT:** Use `SynaraEntity.delete()` (soft delete by default) or log to audit trail before hard deletion.
