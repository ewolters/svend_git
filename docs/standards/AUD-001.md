## **1. SCOPE AND PURPOSE**

### **1.1 Purpose**

AUD-001 defines the audit trail architecture for Kjerne/Svend. The audit system provides tamper-proof, hash-chained logging of security-relevant events with per-tenant isolation, integrity verification, and automated compliance reporting.

**Core Principle:**

> Audit logs are immutable evidence. Every entry is hash-chained to the previous one.
> If anyone tampers with the chain, verification detects it. No exceptions.

### **1.2 Scope**

**Applies to:**
- All API mutations (POST, PUT, PATCH, DELETE) on `/api/` paths
- Authentication events (login, logout, failed attempts)
- Authorization failures
- Data access patterns for compliance-sensitive resources
- Integrity violation detection and escalation

**Does NOT apply to:**
- Application-level logging (see LOG-001)
- Read-only GET requests (unless explicitly configured)
- Static file serving, health checks

## **3. TERMINOLOGY**

| Term | Definition |
|------|------------|
| **Hash chain** | Blockchain-style linkage where each entry's hash includes the previous entry's hash. Tampering with any entry invalidates all subsequent hashes. |
| **Genesis entry** | First entry in a tenant's chain. `previous_hash` is `"0" * 64` (64 zero characters). |
| **Chain head** | Most recent entry in a tenant's chain. |
| **Payload hash** | SHA-256 of the event payload (JSON, sorted keys). |
| **Current hash** | SHA-256 of the full entry including `previous_hash`, creating the chain link. |
| **Integrity violation** | Detected mismatch between stored and recomputed hashes. |
| **Compliance check** | Automated verification of a specific security/compliance control. |

Current hash is computed from:
<!-- code: correct -->
```python
hash_data = {
    "timestamp": self.timestamp.isoformat(),
    "actor": self.actor,
    "event_name": self.event_name,
    "payload_hash": self.payload_hash,           # NOT raw payload
    "correlation_id": str(self.correlation_id),
    "tenant_id": str(self.tenant_id),
    "previous_hash": self.previous_hash,          # CHAIN LINK
}
hash_json = json.dumps(hash_data, sort_keys=True)
current_hash = hashlib.sha256(hash_json.encode("utf-8")).hexdigest()
```

### **4.4 Chain Integrity**

<!-- assert: Hash chain computed atomically with select_for_update to prevent race conditions | check=aud-atomic-chain -->
<!-- impl: syn/audit/utils.py:generate_entry -->
<!-- test: syn.audit.tests.test_syslog.TestGenerateEntry.test_generate_entry_creates_chain -->
<!-- test: syn.audit.tests.test_syslog.TestGenerateEntry.test_generate_entry_basic -->
<!-- test: syn.audit.tests.test_syslog.TestGenerateEntry.test_generate_entry_with_correlation -->

Hash chain is computed inside `transaction.atomic()` with `select_for_update()` to prevent race conditions:

1. Lock previous entries for this tenant
2. Get chain head (most recent entry)
3. Set `previous_hash` from chain head's `current_hash`
4. Compute `current_hash` including `previous_hash`
5. Save atomically

**Location:** `syn/log/middleware.py`

Automatically creates `SysLogEntry` for API mutations.

<!-- assert: AUDIT_PATHS and AUDIT_METHODS configured in Django settings | check=aud-audit-settings -->
<!-- impl: svend/settings.py -->
<!-- test: syn.audit.tests.test_syslog.TestComplianceRequirements.test_audit_trail_completeness -->

**Configuration (Django settings):**
<!-- code: correct -->
```python
AUDIT_PATHS = ["/api/"]                              # Paths to audit
AUDIT_METHODS = ["POST", "PUT", "PATCH", "DELETE"]   # Methods to audit
```

**Behavior:**
1. Check if request matches `AUDIT_PATHS` and `AUDIT_METHODS`
2. Let request process normally
3. If response status < 500, create audit entry with:
   - `actor`: authenticated user email or "anonymous"
   - `event_name`: `api.{method}.{path_as_dots}` (UUIDs stripped)
   - `payload`: method, path, status_code, client_ip, query params (sanitized)
   - `correlation_id`: from request (validated as UUID)
   - `tenant_id`: from request or user

**Location:** `syn/audit/utils.py`

Verifies the entire hash chain for a tenant:

1. Check genesis entry: `is_genesis=True`, `previous_hash="0"*64`
2. For each entry: recompute hash and compare (`verify_hash()`)
3. For each entry: verify chain link to previous (`verify_chain_link()`)
4. Check for gaps in sequential IDs

**Returns:**
```python
{
    "is_valid": True/False,
    "total_entries": 42,
    "violations": [...],
    "genesis_valid": True/False,
    "message": "Chain is intact"
}
```

## **9. ANTI-PATTERNS**

### **9.1 Mutable Audit Records**

<!-- rule: mandatory -->

**PROHIBITED:** Any mechanism that allows updating or deleting `SysLogEntry` records.

Both `save()` (on existing records) and `delete()` raise exceptions. Django admin has no change/delete permissions.

### **9.2 Skipping Correlation ID**

<!-- rule: mandatory -->
<!-- assert: All SysLogEntry records SHOULD have a non-null correlation_id — system warns on missing values | check=aud-correlation-required -->

**PROHIBITED:** Creating audit entries without `correlation_id`.

The system logs a warning when `correlation_id` is missing. All entries SHOULD have a correlation_id linking them to the originating request.

### **9.3 Unverified Chains**

<!-- rule: mandatory -->
<!-- assert: Chain verification MUST run at least daily per tenant to detect tampering | check=aud-daily-verification -->

**PROHIBITED:** Operating in production without periodic chain verification.

Chain verification MUST run at least daily per tenant to detect tampering.

### **9.4 Plaintext Sensitive Data in Payload**

<!-- assert: AuditLoggingMiddleware strips sensitive fields from payload before storage | check=aud-payload-sanitize -->
<!-- impl: syn/log/middleware.py:AuditLoggingMiddleware -->
<!-- test: syn.audit.tests.test_syslog.TestComplianceRequirements.test_audit_trail_completeness -->

<!-- rule: mandatory -->

**PROHIBITED:** Storing passwords, tokens, or encryption keys in `payload`.

The `AuditLoggingMiddleware` strips `password`, `token`, and `key` from query params. Callers of `generate_entry()` must sanitize payloads.
