## **1. SCOPE AND PURPOSE**

### **1.1 Purpose**

ERR-001 defines the error handling architecture for Kjerne/Svend. All infrastructure-level errors use a typed exception hierarchy with correlation tracking, automatic severity-based logging, HTTP status mapping, and structured API error envelopes.

**Core Principle:**

> Every error has a category, a severity, a correlation ID, and a clear HTTP mapping.
> Errors are data — they serialize, they log, they trace. No naked exceptions in production code paths.

### **1.2 Scope**

**Applies to:**
- All `syn/` infrastructure code
- Error responses from `ErrorEnvelopeMiddleware`
- Retry and circuit breaker patterns for external dependencies

**Does NOT apply to:**
- Django's built-in validation (forms, model clean)
- Agent-level error handling in `agents/` (uses its own patterns)
- Frontend JavaScript error handling

### **1.3 Implementation**

| Component | Location | Purpose |
|-----------|----------|---------|
| `SynaraError` + subclasses | `syn/err/exceptions.py` | Typed exception hierarchy |
| Error types & enums | `syn/err/types.py` | ErrorCategory, ErrorSeverity, ErrorEnvelope |
| Retry & circuit breaker | `syn/err/retry.py` | Resilience patterns |
| Error registry | `syn/err/types.py` | Registered error codes with metadata |
| `ErrorEnvelopeMiddleware` | `syn/api/middleware.py` | Wraps unhandled errors in standard envelope |

## **3. TERMINOLOGY**

| Term | Definition |
|------|------------|
| **SynaraError** | Base exception class. All syn/ exceptions inherit from it. |
| **ErrorCategory** | Classification: VALIDATION, AUTHENTICATION, AUTHORIZATION, NOT_FOUND, CONFLICT, RATE_LIMIT, DEPENDENCY, DATABASE, TIMEOUT, SYSTEM. |
| **ErrorSeverity** | DEBUG, INFO, WARNING, ERROR, CRITICAL. |
| **ErrorEnvelope** | Standardized JSON error response format for API consumers. |
| **Circuit breaker** | Pattern that fails fast when a dependency is unavailable: CLOSED → OPEN → HALF_OPEN → CLOSED. |
| **Bulkhead** | Limits concurrent calls to a dependency to prevent resource exhaustion. |
| **Retryable** | Error categories that can be retried: RATE_LIMIT, DEPENDENCY, TIMEOUT. |

## **4. EXCEPTION HIERARCHY**

### **4.1 Base Class**

<!-- assert: All syn/ exceptions inherit from SynaraError base class | check=err-base-hierarchy -->
<!-- impl: syn/err/exceptions.py:SynaraError -->

```python
class SynaraError(Exception):
    message: str                    # Human-readable description
    code: str                       # Machine-readable code (e.g., "validation_error")
    category: ErrorCategory         # Maps to HTTP status
    severity: ErrorSeverity         # Maps to log level
    correlation_id: UUID            # Auto-generated if not provided
    retryable: bool                 # Derived from category
    http_status: int                # Derived from category
    context: ErrorContext | None    # Layer, operation, tenant, user
    details: list[ErrorDetail]      # Field-level errors (validation)
    cause: Exception | None         # Wrapped original exception
```

<!-- assert: SynaraError auto-logs at severity-appropriate level | check=err-auto-logging -->
<!-- impl: syn/err/exceptions.py:SynaraError.__init__ -->

<!-- assert: CRITICAL errors logged to syn.audit logger | check=err-critical-audit -->
<!-- impl: syn/err/exceptions.py:SynaraError.__init__ -->

<!-- assert: correlation_id auto-generated if not provided | check=err-correlation-id -->
<!-- impl: syn/err/exceptions.py:SynaraError.__init__ -->

Every `SynaraError` automatically:
1. Logs to `syn.errors` logger at appropriate level
2. Logs CRITICAL errors to `syn.audit` logger
3. Generates a `correlation_id` if not provided
4. Derives `retryable` and `http_status` from category

## **6. RESILIENCE PATTERNS**

### **6.1 Retry with Exponential Backoff**

<!-- assert: Retry decorator respects max_attempts and only retries retryable categories | check=err-retry-decorator -->
<!-- impl: syn/err/retry.py:retry -->
<!-- impl: syn/err/retry.py:ExponentialBackoff -->

<!-- code: correct -->
```python
from syn.err.retry import retry

@retry(max_attempts=3, base_delay_ms=1000)
def call_external_service():
    return requests.get(url)
```

**Strategies:** NONE, IMMEDIATE, LINEAR, EXPONENTIAL (default), CONSTANT
**Formula:** `delay = min(base_delay * 2^attempt * (1 + jitter), max_delay)`
**Retryable categories:** RATE_LIMIT, DEPENDENCY, TIMEOUT

### **6.2 Circuit Breaker**

<!-- assert: Circuit breaker transitions CLOSED → OPEN → HALF_OPEN → CLOSED | check=err-circuit-breaker -->
<!-- impl: syn/err/retry.py:CircuitBreaker -->
<!-- impl: syn/err/retry.py:with_circuit_breaker -->

<!-- code: correct -->
```python
from syn.err.retry import with_circuit_breaker, RecoveryMode

@with_circuit_breaker("stripe-api", recovery_mode=RecoveryMode.FALLBACK, fallback=lambda: None)
def charge_customer(amount):
    return stripe.charges.create(amount=amount)
```

**States:** CLOSED → OPEN (after 5 consecutive failures) → HALF_OPEN (after 30s) → CLOSED (after 3 successes)
**Config:** `failure_threshold=5`, `success_threshold=3`, `recovery_timeout_ms=30000`
**Recovery modes:** FAIL_FAST, FALLBACK, QUEUE, SKIP

### **6.3 Bulkhead**

<!-- assert: Bulkhead limits concurrent calls and raises BulkheadFullError (429) when full | check=err-bulkhead -->
<!-- impl: syn/err/retry.py:Bulkhead -->

<!-- code: correct -->
```python
from syn.err.retry import Bulkhead

bulkhead = Bulkhead("anthropic-api", max_concurrent=10, max_wait_ms=5000)
with bulkhead.acquire():
    response = call_anthropic()
```

Limits concurrent calls via semaphore. Raises `BulkheadFullError` (429) when full.

### **6.4 Exception Wrapping**

<!-- assert: wrap_exception maps stdlib exceptions to SynaraError subclasses | check=err-wrap-exception -->
<!-- impl: syn/err/exceptions.py:wrap_exception -->

<!-- code: correct -->
```python
from syn.err.exceptions import wrap_exception

try:
    result = external_call()
except Exception as e:
    raise wrap_exception(e, correlation_id=request.correlation_id)
```

Maps: `ValueError` → `ValidationError`, `PermissionError` → `AuthorizationError`, `ConnectionError` → `DependencyError`, `TimeoutError` → `TimeoutError`, else → `SystemError`.

## **7. ANTI-PATTERNS**

### **7.1 Bare Except Clauses**

<!-- rule: mandatory -->

**PROHIBITED:**
<!-- code: prohibited -->
```python
try:
    process()
except:
    pass
```

**CORRECT:**
<!-- code: correct -->
```python
try:
    process()
except SynaraError:
    raise  # re-raise typed errors
except Exception as e:
    raise wrap_exception(e, correlation_id=corr_id)
```

### **7.2 Missing Correlation in Error Responses**

**PROHIBITED:** Returning error JSON without `request_id` or `correlation`.

**CORRECT:** Use `error.to_envelope(request_id=request.syn_request_id)` or let `ErrorEnvelopeMiddleware` handle it.

### **7.3 Retrying Non-Retryable Errors**

**PROHIBITED:** Retrying validation errors (400), auth errors (401/403), or not-found (404).

Only RATE_LIMIT, DEPENDENCY, and TIMEOUT are retryable. The `retryable` field on every error makes this explicit.

### **7.4 Swallowing Circuit Breaker Signals**

**PROHIBITED:** Catching `CircuitBreakerOpenError` and retrying anyway.

The circuit breaker exists to prevent cascading failures. Respect it — use `RecoveryMode.FALLBACK` if you need graceful degradation.
