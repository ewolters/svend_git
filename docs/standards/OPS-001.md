## **1. SCOPE AND PURPOSE**

### **1.1 Purpose**

OPS-001 defines the operational requirements for deploying, running, securing, and maintaining the Svend production environment. This machine is the production server — every operational decision matters.

**Core Principle:**

> The production server is not a development machine that happens to serve traffic.
> Every service, timer, and script has a defined purpose, a restart policy, and a log destination.

### **1.2 Scope**

**Applies to:**
- All files in `services/svend/web/ops/`
- Systemd service and timer definitions
- Gunicorn and Caddy configuration
- Backup and purge scripts
- Django production settings (`svend/settings.py`)
- SSL/TLS and security hardening

**Does NOT apply to:**
- Local development environments
- Agent infrastructure (`agents/`)
- Application-level business logic

## **3. TERMINOLOGY**

| Term | Definition |
|------|------------|
| **Production server** | The machine running svend.ai. This is not a cluster — it is a single node. |
| **Systemd unit** | A service or timer definition managed by systemd. All long-running processes use systemd. |
| **Caddy** | Reverse proxy that handles TLS termination, static files, and security headers. |
| **Gunicorn** | WSGI server running Django with gthread workers behind Caddy. |
| **WhiteNoise** | Django middleware serving compressed, content-hashed static files. |
| **B2** | Backblaze B2 object storage for off-site encrypted backups. |

## **6. BACKUP AND RECOVERY**

### **6.1 Database Backup**

<!-- assert: Encrypted database backups run daily with 30-day retention | check=ops-backup -->
<!-- impl: ops/backup_db.sh -->
<!-- impl: ops/svend-backup.service -->
<!-- impl: ops/svend-backup.timer -->
<!-- test: syn.audit.tests.test_operations.BackupScriptTest.test_backup_script_exists -->
<!-- sla: Backup recovery point objective | metric=durability | target=24h | window=per_incident | severity=critical | measurement=automated | soc2=CC9.2 | check=sla-backup-rpo -->

| Parameter | Value |
|-----------|-------|
| Database | PostgreSQL (`svend`) |
| Schedule | Daily at 03:00 UTC |
| Encryption | AES-256-CBC with PBKDF2 |
| Compression | gzip |
| Key file | `~/.svend_encryption_key` |
| Local retention | 30 days (auto-pruned) |
| Off-site | Backblaze B2 (if configured) |
| Format | `svend_YYYYMMDD_HHMMSS.sql.gz.enc` |

Pipeline: `pg_dump | gzip | openssl enc -aes-256-cbc -pbkdf2 -> file`

### **6.2 Restore Procedure**

```bash
./ops/backup_db.sh --restore <encrypted_file>
# Decrypts -> decompresses -> pipes to psql
```

### **6.3 Off-Site Backup**

B2 upload triggers when `B2_BUCKET` env var is set and `b2` CLI is installed. Credentials in `~/.svend_b2_env`. B2 lifecycle rules handle retention.

## **11. ENCRYPTION AT REST**

### **11.1 Field-Level Encryption**

<!-- assert: Sensitive fields encrypted at rest using Fernet symmetric encryption | check=ops-encryption -->
<!-- impl: svend/settings.py -->
<!-- test: syn.audit.tests.test_operations.EncryptionConfigTest.test_fernet_key_configured -->

Key file: `~/.svend_encryption_key`. Algorithm: Fernet (PBKDF2). Django setting: `FIELD_ENCRYPTION_KEY`. Applied to `EncryptedCharField` model fields (e.g., `stripe_customer_id`).

### **11.2 Backup Encryption**

All database backups encrypted with AES-256-CBC via OpenSSL before storage. Key never leaves the server. See §6.1.

## **12. STARTUP SEQUENCE**

### **12.1 Production Boot**

1. Systemd starts `svend.service` -> `ops/start_prod.sh`
2. Script sources `.env` (SVEND_* variables)
3. Script loads encryption key from `~/.svend_encryption_key`
4. Gunicorn starts with gthread workers, binds `127.0.0.1:8000`
5. Caddy (separate service) listens `:80/:443`, reverse proxies to `:8000`
6. Caddy auto-provisions SSL certificate
7. Tempora (optional) runs in single-node mode on `:7000`

## **13. ANTI-PATTERNS**

### **13.1 Running as Root**

**PROHIBITED:** Running any service as root. All services run as `eric:eric`.

### **13.2 Exposing Gunicorn Directly**

**PROHIBITED:** Binding Gunicorn to `0.0.0.0`. It MUST bind to `127.0.0.1` with Caddy as reverse proxy.

### **13.3 Unencrypted Backups**

**PROHIBITED:** Storing database dumps without encryption. All backups MUST pass through the OpenSSL encryption pipeline.

<!-- code: prohibited -->
```bash
pg_dump svend > backup.sql
```

<!-- code: correct -->
```bash
pg_dump svend | gzip | openssl enc -aes-256-cbc -pbkdf2 \
  -pass file:~/.svend_encryption_key -out svend_$(date +%Y%m%d).sql.gz.enc
```

### **13.4 Disabling Security Headers**

**PROHIBITED:** Removing or weakening security headers in Caddyfile. CSP, HSTS, and X-Frame-Options are mandatory.

### **13.5 Skipping Collectstatic**

**PROHIBITED:** Deploying without running `collectstatic`. WhiteNoise requires the manifest to serve compressed, hashed assets.
