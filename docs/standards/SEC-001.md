## **1. SCOPE AND PURPOSE**

### **1.1 Purpose**

SEC-001 defines the security architecture for Kjerne/Svend. It covers authentication, authorization, tenant isolation, encryption, transport security, rate limiting, and error redaction as implemented across the `syn/`, `accounts/`, and `core/` packages.

**Core Principle:**

> Security is not a layer — it is a property of every layer.
> Every request passes through authentication, authorization, tenant isolation, rate limiting, and error redaction before a response leaves the system. No shortcuts.

### **1.2 Scope**

**Applies to:**
- All API endpoints under `/api/` and `/billing/`
- Session and cookie security configuration
- Tenant-scoped data access and query filtering
- Field-level encryption and secrets management
- Transport security headers (HSTS, CSP, CORS)
- Rate limiting at user and tenant levels
- Error response sanitization and PII redaction

**Does NOT apply to:**
- Static file serving (WhiteNoise — no auth required)
- Health check endpoints
- Agent-internal processing (agents/ has its own patterns)
- Frontend JavaScript security (XSS mitigation is CSP + template escaping)

### **1.3 Implementation Overview**

| Component | Location | Purpose |
|-----------|----------|---------|
| Custom User model | `accounts/models.py` | AbstractUser with email verification |
| Permission decorators | `accounts/permissions.py` | Auth, tier, feature, org-role gates |
| Tier constants | `accounts/constants.py` | Rate limits, feature sets per plan |
| Tenant middleware | `syn/synara/middleware/tenant.py` | Thread-local tenant isolation |
| Tenant models | `core/models/tenant.py` | Tenant, Membership, OrgInvitation |
| Field encryption | `core/encryption.py` | Fernet-based encrypted model fields |

| Bot detection | `accounts/middleware.py` | Crawler, headless, spoofed-UA detection |
| Django settings | `svend/settings.py` | Session, HSTS, SSL, cookie config |

## **3. TERMINOLOGY**

<!-- table: terminology -->

| Term | Definition |
|------|------------|
| **Session-based auth** | Authentication via Django's session framework. Session ID stored in a signed, HttpOnly cookie. No JWTs. |
| **Tenant** | An organizational unit (UUID-identified) that owns shared resources. Users belong to tenants via `Membership`. |
| **Tier** | Subscription level determining rate limits and feature access: FREE, FOUNDER, PRO, TEAM, ENTERPRISE. |
| **Feature gate** | Boolean check against a user's tier to enable/disable platform capabilities. |
| **Fernet** | Symmetric encryption using AES-128-CBC + HMAC-SHA256 (via `cryptography` library). |
| **Envelope encryption** | Two-layer scheme: a Key Encryption Key (KEK) encrypts a Data Encryption Key (DEK), which encrypts the value. |
| **Token hashing** | One-way SHA-256 hash of verification tokens. The raw token is sent to the user; only the hash is stored. |
| **HSTS** | HTTP Strict Transport Security — browser directive to only connect over HTTPS. |
| **CSP** | Content-Security-Policy — browser directive restricting script/resource origins. |
| **Error redaction** | Stripping sensitive patterns (passwords, tokens, PII) from error responses before they leave the server. |
| **Thread-local tenant** | Per-request tenant context stored in Python threading.local, set by middleware, consumed by query mixins. |
| **Bot detection** | User-agent and behavioral analysis to identify crawlers, headless browsers, and spoofed clients. |

The verification flow:

1. **Generate** a raw token: `secrets.token_urlsafe(32)` (256 bits of entropy)
2. **Hash** the token: `hash_token(raw_token)` from `core/encryption.py` produces a SHA-256 hex digest
3. **Store** only the hash in `User.verification_token`
4. **Send** the raw token to the user via email
5. **Verify** by hashing the submitted token and comparing to the stored hash

<!-- code: correct -->
```python
import secrets
from core.encryption import hash_token

raw_token = secrets.token_urlsafe(32)
user.verification_token = hash_token(raw_token)
user.verification_token_created = timezone.now()
user.save()
# Send raw_token in verification email
```

**Rules:**
- Raw tokens MUST NOT be stored in the database
- Token comparison MUST be hash-to-hash (timing-safe comparison via `hmac.compare_digest` is recommended)
- Tokens SHOULD have an expiry window checked against `verification_token_created`

### **6.1 TenantIsolationMiddleware**

<!-- assert: TenantIsolationMiddleware sets thread-local tenant for every authenticated request | check=sec-tenant-middleware -->
<!-- impl: syn/synara/middleware/tenant.py:TenantIsolationMiddleware -->
<!-- rule: mandatory -->
<!-- test: syn.audit.tests.test_security.TenantMiddlewareTest.test_tenant_middleware_registered -->

The middleware executes on every request and:

1. **Skips** exempt paths (static files, health checks, public endpoints)
2. **Bypasses** for staff users (Django admin access)
3. **Resolves** the tenant from the authenticated user's membership
4. **Sets** the thread-local tenant via `set_current_tenant(tenant)`
5. **Clears** the thread-local tenant after the response (in `finally` block)

<!-- code: correct -->
```python
# In middleware process_request:
tenant = resolve_tenant(request.user)
set_current_tenant(tenant)

# In middleware process_response / finally:
set_current_tenant(None)  # Always clear
```

<!-- impl: syn/synara/middleware/tenant.py:TenantQuerySetMixin -->
<!-- test: syn.core.tests.test_base_models.TestSynaraEntityManager.test_for_tenant_filters_by_tenant_id -->

Models that use `TenantQuerySetMixin` (or inherit from a tenant-aware base) automatically scope all queries to the current thread-local tenant:

<!-- code: correct -->
```python
class TenantQuerySetMixin:
    def get_queryset(self):
        qs = super().get_queryset()
        tenant = get_current_tenant()
        if tenant:
            qs = qs.filter(tenant=tenant)
        return qs
```

This prevents cross-tenant data leakage at the ORM level — views never see data from other tenants.

The secrets management system uses two-layer envelope encryption:

```
KEK (Key Encryption Key)

                  └── encrypts → Secret Value
```

### **7.3 Token Hashing**

<!-- assert: hash_token() uses SHA-256 for one-way token storage | check=sec-token-hashing -->
<!-- impl: core/encryption.py:hash_token -->
<!-- test: syn.core.tests.test_secrets.SecretStoreTestCase.test_set_and_get_secret -->
<!-- test: syn.core.tests.test_secrets.SecretStoreTestCase.test_secret_not_found -->

<!-- rule: mandatory -->

All tokens (verification, password reset, API keys for lookup) are stored as SHA-256 hashes:

<!-- code: correct -->
```python
import hashlib

def hash_token(token: str) -> str:
    return hashlib.sha256(token.encode("utf-8")).hexdigest()
```

**Rules:**
- NEVER store raw tokens in the database
- Use `hash_token()` for all token storage — do not hand-roll hashing
- For API key lookup, store the hash; for API key retrieval, use `EncryptedCharField`

## **12. ANTI-PATTERNS**

### **11.1 Raw Token Storage**

<!-- rule: mandatory -->

**PROHIBITED:** Storing raw verification tokens, API keys, or session secrets in the database.

<!-- code: prohibited -->
```python
user.verification_token = raw_token  # WRONG: raw token stored
user.save()
```

**CORRECT:**
<!-- code: correct -->
```python
user.verification_token = hash_token(raw_token)  # Only the hash is stored
user.save()
```

### **11.2 Missing Permission Decorators**

<!-- rule: mandatory -->

**PROHIBITED:** API views without authentication or authorization decorators.

<!-- code: prohibited -->
```python
@csrf_exempt
def sensitive_endpoint(request):
    # No auth check — anyone can access
    return JsonResponse(get_private_data())
```

**CORRECT:**
<!-- code: correct -->
```python
@csrf_exempt
@require_auth
@rate_limited
def sensitive_endpoint(request):
    return JsonResponse(get_private_data())
```

### **11.3 Direct Tenant Filtering**

<!-- rule: mandatory -->

**PROHIBITED:** Manually filtering by tenant in views instead of using the query mixin.

<!-- code: prohibited -->
```python
def my_view(request):
    # Manual filter — easy to forget, inconsistent
    items = Item.objects.filter(tenant_id=request.user.tenant_id)
```

**CORRECT:**
<!-- code: correct -->
```python
# Use TenantQuerySetMixin on the model manager
# The middleware + mixin handle filtering automatically
def my_view(request):
    items = Item.objects.all()  # Already tenant-scoped via mixin
```

### **11.4 Unredacted Error Details**

<!-- rule: mandatory -->

**PROHIBITED:** Returning raw exception messages to API consumers.

<!-- code: prohibited -->
```python
except Exception as e:
    return JsonResponse({"error": str(e)}, status=500)
    # May leak DB connection strings, file paths, credentials
```

**CORRECT:** Let `ErrorEnvelopeMiddleware` handle it, or use `redact_error_message()`:

<!-- code: correct -->
```python
from syn.api.middleware import redact_error_message

except Exception as e:
    raise wrap_exception(e, correlation_id=request.correlation_id)
    # ErrorEnvelopeMiddleware wraps and redacts automatically
```

### **11.5 Encryption Key in Source Code**

<!-- rule: mandatory -->

**PROHIBITED:** Hardcoding `FIELD_ENCRYPTION_KEY` or any secret in Python source files.

**CORRECT:** All encryption keys and secrets MUST come from environment variables. The `FIELD_ENCRYPTION_KEY` setting reads from `os.environ`.

### **11.6 Wildcard CORS in Production**

<!-- rule: mandatory -->

**PROHIBITED:** Setting `CORS_ALLOW_ALL_ORIGINS = True` in production.

**CORRECT:** Explicitly list allowed origins in `CORS_ALLOWED_ORIGINS`.
