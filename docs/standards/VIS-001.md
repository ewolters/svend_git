## **1. SCOPE AND PURPOSE**

### 1.1 Purpose

Define mandatory visual consistency rules for every chart the Svend platform produces. Every chart must look like it came from the same product — same colors, same legend placement, same backgrounds, same typography.

**Core Principle:**

> A user should never be able to tell which module generated a chart by looking at its styling. Only the data and chart type should vary.

### 1.2 Scope

**Applies to:**
- All Plotly chart dicts returned by DSW analysis engines (stats, SPC, ML, Bayesian, viz, simulation)
- All SPC control chart and capability study charts
- All DOE (experimenter) result charts
- All forecast, reliability, and exploratory analysis charts

**Does not apply to:**
- Chart.js charts in the internal dashboard (governed by FE-001 §6.4)
- SVG export rendering (chart_render.py — uses white backgrounds for print)
- Third-party embedded visualizations (SmilesDrawer chemistry diagrams)

### 1.3 Enforcement Architecture

All chart output passes through `standardize_output()` (in `agents_api/dsw/standardize.py`) which calls `apply_chart_defaults()` (in `agents_api/dsw/chart_defaults.py`). This is the single enforcement point — rules are applied at the pipeline layer, not in individual analysis modules.

## **3. TERMINOLOGY**

- **Plot dict** — A Python dict with `data` (list of traces) and `layout` (Plotly layout config)
- **Trace** — A single data series in a Plotly chart (scatter, bar, histogram, box, heatmap)
- **SVEND_COLORS** — 10-color categorical palette defined in `common.py`
- **Semantic colors** — Named constants (COLOR_GOOD, COLOR_BAD, etc.) for consistent meaning
- **apply_chart_defaults()** — Pipeline function that enforces VIS-001 on every chart

## **4. COLOR PALETTE**

### 4.1 Trace Colors

<!-- assert: All chart trace colors MUST be from SVEND_COLORS or the semantic color constants defined in common.py -->
<!-- impl: agents_api/dsw/common.py:SVEND_COLORS -->
<!-- impl: agents_api/dsw/chart_defaults.py:_apply_trace_colors -->
<!-- test: agents_api.tests.test_vis_compliance.ColorPaletteTest -->

All chart trace colors (marker.color, line.color, fillcolor) MUST resolve to one of:

**Categorical palette** (SVEND_COLORS, 10 slots):

| Slot | Hex | CSS Variable | Use |
|------|-----|-------------|-----|
| 0 | `#4a9f6e` | --accent-primary | Primary series |
| 1 | `#4a9faf` | --accent-blue | Secondary series |
| 2 | `#e89547` | --accent-orange | Tertiary series |
| 3 | `#e8c547` | --accent-gold | Quaternary series |
| 4 | `#8a7fbf` | --accent-purple | 5th series |
| 5 | `#d66b9f` | — | 6th series |
| 6 | `#4ac9c0` | — | 7th series |
| 7 | `#c97a4a` | — | 8th series |
| 8 | `#6b9f4a` | — | 9th series |
| 9 | `#5b8bd6` | — | 10th series |

**Semantic colors** (for consistent meaning):

| Constant | Hex | Meaning |
|----------|-----|---------|
| COLOR_GOOD | `#4a9f6e` | Success, in-control, healthy |
| COLOR_BAD | `#d06060` | Error, out-of-control, defect |
| COLOR_WARNING | `#e89547` | Caution, moderate |
| COLOR_INFO | `#4a9faf` | Informational, secondary |
| COLOR_NEUTRAL | `#888888` | Noise floor, baseline, inactive |
| COLOR_REFERENCE | `#d4a24a` | Reference lines, pooled values |
| COLOR_GOLD | `#e8c547` | Tertiary emphasis, threshold |

Alpha variants of any allowed color (via `_rgba()`) are permitted.

### 4.2 Off-Palette Color Normalization

<!-- assert: Off-palette hex colors in chart traces MUST be normalized to the nearest palette equivalent by apply_chart_defaults -->
<!-- impl: agents_api/dsw/chart_defaults.py:_normalize_color -->
<!-- test: agents_api.tests.test_vis_compliance.ColorPaletteTest.test_normalization -->

`apply_chart_defaults()` maintains a normalization map that converts known off-palette colors to their palette equivalents. This is the architectural enforcement mechanism — analysis modules do not need modification.

### 4.3 Alpha Variants

Alpha/transparency variants of palette colors SHOULD use the `_rgba()` helper from `common.py` for consistency. Raw `rgba()` strings are acceptable in output (WARN, not FAIL) as long as the base RGB values correspond to a palette color.

## **5. LEGEND PLACEMENT**

### 5.1 Standard Position

<!-- assert: Chart legends MUST use horizontal orientation below the chart per LEGEND_DEFAULTS unless the chart type requires an exception -->
<!-- impl: agents_api/dsw/chart_defaults.py:LEGEND_DEFAULTS -->
<!-- test: agents_api.tests.test_vis_compliance.LegendPositionTest -->

The canonical legend configuration is:

```python
LEGEND_DEFAULTS = {
    "orientation": "h",      # horizontal
    "yanchor": "bottom",
    "y": -0.25,              # below chart
    "xanchor": "left",
    "x": 0,
}
```

### 5.2 Inside-Chart Prohibition

<!-- assert: Legends MUST NOT be positioned inside the chart body (0 < y < 1 with non-horizontal orientation) -->
<!-- impl: agents_api/dsw/chart_defaults.py:apply_chart_defaults -->
<!-- test: agents_api.tests.test_vis_compliance.LegendPositionTest.test_no_legend_inside_chart -->

Legends positioned inside the chart area obscure data and are inconsistent across analyses. `apply_chart_defaults()` detects and replaces inside-chart legends with the standard position.

**Acceptable positions:** Below chart (y < 0), above chart (y > 1 with horizontal orientation).
**Prohibited:** Inside chart body (0 < y < 1) with vertical or default orientation.

## **6. BACKGROUND AND TRANSPARENCY**

### 6.1 Transparent Backgrounds

<!-- assert: paper_bgcolor and plot_bgcolor MUST be rgba(0,0,0,0) for theme compatibility -->
<!-- impl: agents_api/dsw/chart_defaults.py:apply_chart_defaults -->
<!-- test: agents_api.tests.test_vis_compliance.BackgroundTest -->

All interactive charts use transparent backgrounds so they adapt to the active theme (dark, light, nordic, sandstone, midnight, contrast). The frontend applies theme-appropriate backgrounds via CSS.

<!-- code: correct -->
```python
layout["paper_bgcolor"] = "rgba(0,0,0,0)"
layout["plot_bgcolor"] = "rgba(0,0,0,0)"
```

<!-- code: prohibited -->
```python
layout["paper_bgcolor"] = "#1a1a2e"   # hardcoded dark background
layout["plot_bgcolor"] = "#0a0f0a"    # hardcoded dark background
```

## **10. SEMANTIC COLOR USAGE**

### 10.1 Meaning Consistency

Colors with semantic meaning SHOULD use the named constants consistently:

| Meaning | Constant | Use Cases |
|---------|----------|-----------|
| Good / Pass / In-control | COLOR_GOOD | Control chart CL, pass indicators, healthy values |
| Bad / Fail / Out-of-control | COLOR_BAD | UCL/LCL violations, OOC points, failure indicators |
| Warning / Caution | COLOR_WARNING | Zone B in control charts, moderate values |
| Reference / Threshold | COLOR_REFERENCE | Specification limits, target lines, pooled values |
| Neutral / Baseline | COLOR_NEUTRAL | Noise floor, inactive series, grid emphasis |

## **11. POST-PROCESSING PIPELINE**

### 11.1 Mandatory Pipeline

<!-- assert: All chart output MUST pass through apply_chart_defaults() before reaching the client -->
<!-- impl: agents_api/dsw/chart_defaults.py:apply_chart_defaults -->
<!-- impl: agents_api/dsw/standardize.py:standardize_output -->
<!-- test: agents_api.tests.test_vis_compliance.PostProcessingTest -->

The enforcement architecture:

```
Analysis module → result dict with plots[]
    → standardize_output() [standardize.py]
        → apply_chart_defaults(plot) for each plot [chart_defaults.py]
            → _apply_trace_colors() — palette enforcement + normalization
            → legend position enforcement
            → background, margins, font defaults
    → JsonResponse to client
```

**All code paths** that return chart output MUST route through this pipeline. Any endpoint that returns Plotly dicts without calling `standardize_output()` or `apply_chart_defaults()` is a pipeline gap and a FAIL violation.

## **12. ANTI-PATTERNS**

<!-- code: prohibited -->
```python
# WRONG: Hardcoded hex not in palette
"marker": {"color": "#d63031"}

# WRONG: Legend inside chart body
"legend": {"x": 0.98, "y": 0.98, "xanchor": "right"}

# WRONG: Non-transparent background
"paper_bgcolor": "#0a0f0a"

# WRONG: Custom font
"font": {"family": "Arial", "size": 14}

# WRONG: Returning plots without pipeline
return JsonResponse({"plots": [plot_dict]})  # bypasses apply_chart_defaults
```

<!-- code: correct -->
```python
# CORRECT: Use palette constants
from .common import COLOR_BAD, SVEND_COLORS, _rgba
"marker": {"color": COLOR_BAD}
"marker": {"color": SVEND_COLORS[2]}
"fillcolor": _rgba(COLOR_GOOD, 0.3)

# CORRECT: Let apply_chart_defaults set legend (or use LEGEND_DEFAULTS)
from .chart_defaults import LEGEND_DEFAULTS
"legend": dict(LEGEND_DEFAULTS)

# CORRECT: Route through pipeline
from .standardize import standardize_output
result = standardize_output(result, analysis_type, analysis_id)
```
