Working Paper — February 2026

Probabilistic Bayesian SPC

A Unified Framework for Common Cause Capability
and Special Cause Escape Dynamics
Working Draft
Abstract
We introduce Probabilistic Bayesian SPC (PBS), a two-layer framework that regrounds Statistical Process Control in Bayesian inference and stochastic dynamical systems theory. PBS distinguishes two fundamentally different questions classical SPC conflates: (1) How confident are we that the process is capable of not producing defects? — answered by the Bayesian SPC layer through posterior capability distributions; and (2) Is the process undergoing a state transition toward a failure mode? — answered by the Bayesian Escape layer through quasipotential landscape analysis and real-time momentum tracking. The first layer addresses common cause variation through systemic improvement. The second addresses special cause events through root cause investigation. Classical SPC methods — Shewhart charts, CUSUM, EWMA, capability indices — are recovered as limiting cases. We define the Bayesian Sigma metric as a principled successor to the classical sigma level, and establish robustness guarantees under model misspecification.

Foundations
The Problem and the Architecture

1. The Two Questions of Process Control

Consider two kinds of catastrophic events.

A plane crash is a special cause event. Something specific went wrong — a sensor failed, a maintenance procedure was skipped, a design flaw was triggered by an unusual condition. The response is investigation: the NTSB reconstructs the causal chain, identifies the root, and issues corrective action. The logic is forensic. You find the broken link.

A car crash, in the aggregate, is a common cause phenomenon. No single crash has one root cause that, if fixed, prevents all future crashes. Crashes are multivariate, stochastic, and emergent from the interaction of road design, vehicle engineering, driver behavior, and environmental conditions. The response is systemic improvement: better roads, safer cars, improved signage. The logic is kaizen. You improve the process that produces the outcomes.

Figure 1. The two questions of process control. Special cause analysis yields a scalar diagnostic (escape probability). Common cause analysis yields a posterior distribution over process capability. Classical SPC conflates both into control limits.
Core Distinction
Special cause: Go after the root. The process changed; find out why and reverse it.
Common cause: Attack variation itself. The process didn't change; make it better.

Classical SPC knows this distinction — Shewhart drew it in 1931. But classical SPC handles both with the same tool: control charts that flag points outside limits. This conflates two fundamentally different questions:

Question 1 (Common Cause / Capability)
What is our confidence that the process, operating normally, is capable of meeting requirements? A question about the generating distribution itself. The answer is a posterior belief about the parameters that govern process behavior.
Question 2 (Special Cause / Escape)
Is the process currently undergoing a state transition toward a failure mode? A question about dynamics. The answer is a trajectory diagnostic on the quasipotential landscape.

Question 1 produces a distribution. Question 2 produces a scalar (or triad). Question 1 informs capital allocation and long-term strategy. Question 2 informs immediate intervention and root cause investigation. They are different in kind, not degree. PBS treats them as such.

2. The PBS Architecture

Definition (Probabilistic Bayesian SPC)
PBS is a two-layer framework:

Layer 1 — Bayesian SPC: Posterior distribution over process parameters → posterior capability distributions. Answers: "How confident are we?" Addresses common cause. Instrument: kaizen.

Layer 2 — Bayesian Escape: Stochastic dynamical system with metastable states → quasipotential landscape → real-time escape detection. Answers: "Is the process changing?" Addresses special cause. Instrument: root cause investigation.
Figure 2. PBS two-layer architecture. Layer 1 estimates process capability as a posterior and provides ε to Layer 2. Layer 2 detects special cause state transitions on the quasipotential landscape. Root cause findings feed back to update the structural model.
The Inversion
Classical SPC: "What is the probability of a defect?" — a scalar.
PBS: "What is our confidence that the distribution from which we derive defect probabilities is itself adequate?" — a distribution over distributions.

One answers "what's the probability the event will happen." The other answers "what is our confidence in the distribution from which we're deriving those probabilities."

3. From Point Estimates to Posterior Distributions

3.1 The Classical Problem

Classical capability analysis computes a point estimate — Cpk = 1.33 — and treats it as fact. But with 30 observations, the 95% confidence interval on Cpk = 1.33 spans roughly [0.9, 1.7]. The point estimate is nearly meaningless; the uncertainty is the message.

3.2 The Bayesian Model

Definition (Process Parameter Posterior)
Let θ = (μ, σ², ν, ...) be process parameters. Given observations y = (y₁, ..., yn):
p(θ | y) ∝ L(y | θ) · π(θ)

A complete probability distribution over the parameters, conditioned on everything observed.

3.3 Prior Specification

Uninformative: μ ∼ 𝒩(target, large variance), σ ∼ Half-Cauchy(0, scale). Data dominates quickly.

Informative: Historical data encoded — μ ∼ 𝒩(μ̂hist, σ̂²hist/nhist). New data updates existing knowledge.

Structural: Physics constraints as architectural priors. Positivity → log-normal. Known noise floor → truncated σ prior.

4. Bayesian Capability Analysis

4.1 Posterior Capability Distribution

Definition (Posterior Capability)
Propagate the parameter posterior through the capability formula:
Cpk(θ) = min( (USL − μ) / 3σ , (μ − LSL) / 3σ )

For each draw θ(s) from p(θ | y), compute Cpk(s). The collection is the posterior capability distribution.

Figure 3. Posterior distribution of Cpk from n=30 observations. The classical point estimate (dashed line) obscures the full uncertainty. The shaded regions show probability mass at different capability thresholds. With this data, P(Cpk > 1.00) ≈ 3% — the process is not capable, and the posterior makes this unambiguous.

4.2 The Probability of Capability

Definition (Probability of Capability)
P(Cpk > c* | y) = ∫ 𝕀[Cpk(θ) > c*] · p(θ | y) dθ

Not "Cpk = 1.33" but "P(Cpk > 1.33) = 72%". The first is a statistic. The second is actionable belief.

4.3 Multi-Threshold Assessment

P(Cpk > 1.00),   P(Cpk > 1.33),   P(Cpk > 1.50),   P(Cpk > 1.67),   P(Cpk > 2.00)

Each threshold corresponds to a different industry standard. The posterior probability at each level tells the operator exactly how much confidence the data supports at each tier.

5. The Confidence Inversion

Theorem (The Confidence Inversion)
Let D(θ) be any derived quantity. Classical SPC reports D(θ̂) — a scalar of a point estimate. PBS reports p(D | y) — the full posterior of D.

Classical: condition on fixed (estimated) process.
PBS: marginalize over uncertainty in the process itself.

Classical defect rate: a point. PBS posterior predictive defect rate: a distribution with credible intervals and full shape.

Classical:   p̂defect = P(Y ∉ spec | μ̂, σ̂)     [scalar]

PBS:   pdefect(y) = ∫ P(Y ∉ spec | θ) · p(θ | y) dθ     [distribution]

6. Sequential Updating and the Living Capability Posterior

6.1 Online Bayesian Updating

p(θ | y₁,...,yn, yn+1) ∝ L(yn+1 | θ) · p(θ | y₁,...,yn)

Yesterday's posterior is today's prior. The capability distribution is alive — it sharpens with each observation, drifts if the process drifts, widens if variability increases.

Figure 4. Sequential Bayesian updating of the Cpk posterior. At n=5 (light), the posterior is wide — little confidence. By n=30 (medium), it sharpens. By n=100 (dark), uncertainty is small. The posterior is alive: each observation refines belief. Classical capability studies produce a single snapshot; PBS produces a continuous assessment.

6.2 Value of Information

The expected posterior variance of Cpk given m additional observations is computable by forward simulation, yielding a value-of-information curve: marginal uncertainty reduction per observation. When the curve flattens, stop sampling.

6.3 Change Detection in the Capability Posterior

If the capability posterior shifts — median Cpk trending down, or posterior variance increasing without more data — something has changed. This is a soft signal of a potential special cause, delivered before any single observation triggers a control chart. Layer 1 hands this signal to Layer 2 for escape analysis.

7. Stochastic Process Model

7.1 Process Dynamics

Definition (Process SDE)
Let X(t) ∈ ℝd evolve according to:
dX(t) = b(X(t)) dt + √ε · σ(X(t)) dW(t)

where ε > 0 is the noise intensity — the same ε that Layer 1 estimates from data.

Remark. X(t) is the full process state: temperatures, pressures, feed rates, tool wear, environment. The quality characteristic Y(t) = h(X(t)) is an observation function.

7.2 Metastable Equilibria

Definition (Metastable State)
A point x* ∈ ℝd is metastable if: (i) b(x*) = 0, (ii) all eigenvalues of Db(x*) have strictly negative real parts, (iii) the basin B(x*) is bounded.

In-control state = x₀*. Other equilibria = failure modes. The separatrix between basins is the tipping boundary.

8. Large Deviation Theory and the Quasipotential

8.1 Freidlin–Wentzell Framework

Definition (Action Functional)
ST(φ) = ½ ∫₀ᵀ ‖ a(φ(t))⁻¹ [φ̇(t) − b(φ(t))] ‖² dt
Definition (Quasipotential)
V(x*, y) = inf { ST(φ) : φ(0) = x*, φ(T) = y, T > 0 }
Figure 5. The quasipotential landscape for a double-well process. The in-control state (left well) sits at depth Vsaddle below the escape barrier. The minimum action path (MAP) traces the most probable escape trajectory through the saddle point. The process must "climb" the quasipotential hill to transition to the failure mode (right well). The barrier height divided by noise intensity ε determines the expected escape time exponentially.

8.2 Kramers' Law

Theorem (Kramers–Freidlin–Wentzell)
𝔼[τescape] ∼ C · exp(Vsaddle / ε)    as ε → 0

Escape overwhelmingly occurs via the minimum action path (MAP).

9. Escape Dynamics (Forgoston Framework)

9.1 The Escape Hamiltonian

H(x, p) = pTb(x) + ½ pTa(x)p
ẋ = ∂H/∂p = b(x) + a(x)p      ṗ = −∂H/∂x

9.2 Momentum as Risk Indicator

Proposition (Momentum–Risk Correspondence)
Along the MAP: p(t) = ∇V(x₀*, x(t)). The magnitude ‖p‖ measures escape progress.

Remark. Shewhart: "observation is Nσ from center." Momentum: "process has completed X% of the most dangerous escape trajectory."

9.3 Multi-Path Escapes

P(escape via k) / P(escape via j) ∼ exp(−(Vk − Vj) / ε)

Failure modes ranked by escape cost. The escape path through the Hamiltonian system is the causal chain — mathematical foundation for DAG-based root cause analysis.

10. Bayesian State Estimation on the Escape Landscape

10.1 Observation and Filtering

Yn = h(X(tn)) + ηn ,    ηn ∼ 𝒩(0, R)
πn(x) ∝ g(Yn | x) ∫ p(x | x′) πn−1(x′) dx′

10.2 The Bayesian Sigma Triad

Escape Proximity
Δn = 𝔼πn[ Vsaddle − V(x₀*, X) ]
Escape Probability
Pesc(n, τ) ≈ 1 − exp(−τ · λesc(x̂n))
Momentum Alignment
𝒜n = 𝔼πn[ cos ∠(vobs, pMAP(X)) ]

Momentum alignment 𝒜 (gold, middle) rises as trajectory aligns with MAP. Combined ΣB (red, bottom) drops below intervention thresholds well before a Shewhart chart would trigger (dashed line = 3σ rule trigger point).

11. The Bayesian Sigma Metric

Definition (Bayesian Sigma Level)
ΣB(n) = √(2 V(x₀*, x̂n) / ε) · √(1 − 𝒜n⁺)
Theorem (Classical Recovery)
With b = −k(x−μ), σ = σI, h = identity:
V(μ, x) = k‖x − μ‖² / σ²   ⟹   ΣB ∝ ‖x̂ − μ‖ / σ   (classical Z-score)

12. How the Layers Connect

12.1 ε Is the Bridge

ε(n) = 𝔼[σ² | y₁,...,yn]   →   Vsaddle / ε(n) = escape barrier height

Rising variance in Layer 1 → lower escape barriers in Layer 2 → elevated escape probability → earlier intervention. The layers amplify each other's signals.

12.2 The Feedback Loop

Layer Coupling Dynamics
Layer 1 → Layer 2 (forward): Capability posterior provides ε and process parameters defining the quasipotential landscape.

Layer 2 → Layer 1 (feedback): Root cause findings update the structural model — change b(x). Layer 1 re-estimates capability on the modified process.

Kaizen (L1 internal): Reduces ε or reshapes basin — builds inherent capability.
Root cause (L2 → model): Changes topology — eliminates escape paths, raises saddles.

13. Dual State-Space Formulation

Physical State Space (Primary)
X(t) ∈ ℝd = full process physics. SDE, quasipotential, escape paths, Hamiltonian dynamics operate here.
Abstract Quality State Space (Projected)
Q(t) = Π(X(t)) ∈ {q₀,...,qK} by basin membership. Induced dynamics = continuous-time Markov chain.
Theorem (Projection Consistency)
Λij ∼ Cij · exp(−V(xi*, sij) / ε)    for i ≠ j

The abstract chain is derived, not assumed. When physics unknown, use empirical rates (HMM fallback).

Part IV
Robustness, Implementation & Extensions

15. Robustness and Identifiability

PBS Layer 2 rests on the quasipotential V, which depends on the drift field b(x) and diffusion σ(x). If these are misspecified, ΣB inherits the error. This section establishes robustness bounds and degradation guarantees under model uncertainty.

15.1 Sensitivity of ΣB to Drift Perturbation

Theorem (Quasipotential Stability)
Let b(x) be the true drift and b̃(x) = b(x) + δb(x) be a perturbed drift with ‖δb‖∞ ≤ η. Let V and Ṽ be the corresponding quasipotentials. Then for the isotropic diffusion case (a = σ²I):
|V(x*, y) − Ṽ(x*, y)| ≤ (2η / σ²) · L(x*, y) · (D + η·L / 2σ²)

where L(x*, y) is the arc length of the MAP and D = max ‖φ̇ − b(φ)‖ along the path. The quasipotential perturbation is bounded linearly in the drift perturbation magnitude η for small η. This means ΣB degrades gracefully under drift misspecification, not catastrophically.

Proposition (ΣB Sensitivity)
The Bayesian Sigma level satisfies:

|ΣB − Σ̃B| ≤ (1/√(2ε)) · |V − Ṽ|½ · (√V + √Ṽ)⁻¹ + O(momentum correction)

For moderate quasipotential values (not near zero), ΣB is Lipschitz in V, and therefore Lipschitz in the drift perturbation.

Remark. This is the answer to the critique "if V is wrong, ΣB is wrong": yes, but ΣB is continuously wrong, not discontinuously wrong. A 10% error in drift estimation produces a bounded, predictable error in ΣB, not a qualitative failure of the diagnostic. This is a strictly better situation than classical SPC, where the control limits are correct under the Gaussian assumption and meaningless otherwise.

15.2 Bayesian Model Averaging for the Drift

When the drift is uncertain, PBS does not require committing to a single b(x). Instead, maintain a posterior over drift models:

p(b | y)   →   ΣB(y) = ∫ ΣB(n; b) · p(b | y) db

This marginalizes over drift uncertainty, producing a model-averaged Bayesian Sigma that automatically inflates when drift knowledge is poor (because different models predict different V, spreading the posterior on ΣB) and tightens as drift is learned. This is the honest answer: uncertainty about the model propagates into uncertainty about the diagnostic.

For 8–20D physical state spaces, Reduced PBS applies: use gMAM for the MAP computation (which does not require gridding), neural quasipotential for V evaluation, and project to a lower-dimensional manifold identified via PCA or diffusion maps on the particle cloud. The framework does not require solving the full curse of dimensionality — it requires identifying the escape-relevant subspace, which is typically low-dimensional even when the ambient state is not.

15.4 Identifiability Conditions

Theorem (Structural Identifiability)
The PBS framework is identifiable (i.e., distinct parameter values produce distinct observable behavior) under:

(i) The observation function h is injective on each basin (distinct physical states produce distinct quality measurements within a regime);
(ii) The diffusion tensor a(x) is positive definite on the escape-relevant subspace;
(iii) At least two metastable states exist with distinct observational signatures.

Under these conditions, the quasipotential V is identifiable from sufficiently long observation sequences, and the transition rates Λij of the projected chain are consistently estimable.

When identifiability conditions are violated — the observation function cannot distinguish between states, or the noise is degenerate in the escape direction — Layer 2 should report this as an observability limitation rather than producing misleading ΣB values. This is a feature, not a failure: the framework knows when it doesn't know.

16. Computational Architecture

16.1 Layer 1: MCMC / Variational Inference

Capability posterior via NUTS for full exploration or variational inference for real-time. Conjugate priors (Normal-Inverse-Gamma) admit closed-form updates. Non-conjugate models use cached MCMC draws.

16.2 Layer 2: Particle Filter on Quasipotential

Offline: (a) Ordered Upwind for d ≤ 4, (b) gMAM for higher dimensions, (c) Neural Vθ for learned representations.

Online: Particle filter with escape-informed proposal. N=1000, milliseconds per observation on modern GPU hardware.

16.3 Reduced-Order Escape Models

For high-dimensional state spaces (d > 6): identify the escape-relevant subspace via diffusion maps or PCA on the particle cloud near the separatrix. Project escape dynamics onto this subspace. The MAP lies in a low-dimensional manifold even when the ambient space is large — escapes are structured, not random walks in high dimensions.

17. Open Problems

17.1 Adaptive Noise Estimation

Joint estimation of X(t) and ε(t). Hierarchical model where the landscape evolves — valleys shallow as noise grows.

17.2 Non-Markovian Extensions

Stochastic delay differential equations, or augmented state embedding to convert memory to higher-dimensional Markovian.

17.3 Learning Drift from Data

SINDy or neural ODEs. Learned SDE = physics-informed digital twin. Fine-tune language models to generate SDE specs from process descriptions.

17.4 Conformal Prediction Hybrid

Bayesian for insight, conformal for guarantees. Wrap escape probability in conformal bands for distribution-free finite-sample coverage.

17.5 Anytime-Valid Inference

Calibrate Pesc as an e-process. Product of e-values controls false alarm rate under arbitrary stopping.

17.6 Multi-Characteristic PBS

Vector-valued quality characteristics with correlated posteriors and shared escape landscapes.

17.7 FMEA Replacement

Escape path through Hamiltonian = causal chain. Action barrier Vk at each saddle = physics-derived severity × occurrence. Replaces subjective 1–10 FMEA ratings with process dynamics.

17.8 Empirical Validation

The critical next step: a worked example on a synthetic double-well process with asymmetric escape paths, slowly increasing noise, and partial observation. The benchmark: does ΣB give earlier, more calibrated warnings than Shewhart, CUSUM, and EWMA? Simulation study design is specified; execution is forthcoming.
