Frontier Methods for Process Monitoring

E-Detectors, Robust BOCPD, and Conformal Multivariate Monitoring
Eric — Synara Inc. — Working Draft, February 2026
Abstract
This document specifies three frontier statistical methods for implementation in SVEND's quality analytics platform. Part I: E-Detectors — nonparametric sequential changepoint detection using e-processes (Shin, Ramdas & Rinaldo, 2022/2024), providing anytime-valid guarantees without distributional assumptions. Part II: Robust BOCPD — generalized Bayesian online changepoint detection using β-divergence scoring (Altamirano, Briol & Knoblauch, ICML 2023), making the existing BOCPD engine robust to outliers. Part III: Conformal Multivariate Monitoring — extending Burger's (2025) conformal prediction framework to multivariate process monitoring via anomaly detection wrapping. Each part contains: mathematical specification, algorithmic pseudocode, Django data model, chart specification, and anti-patterns. These methods share a common philosophy: distribution-free guarantees, sequential validity, and uncertainty as first-class output.

Part I
E-Detectors: Nonparametric Sequential Changepoint Detection

1. Motivation and Relationship to Existing Tools

SVEND currently has two changepoint-adjacent tools that occupy distinct roles:

Existing ToolWhat It DoesLimitation
BOCPD (Belief Chart)Bayesian posterior over run lengths. Answers: "has the generative model changed?"Parametric (Normal-Gamma conjugate). Sensitive to outliers. No formal false alarm guarantee.
E-value / AVI TestSequential evidence accumulation against H₀. Answers: "is the process shifted from reference?"Tests a fixed null. Does not detect when a change occurred. Not a changepoint detector.

E-Detectors fill the gap: they are changepoint detectors (like BOCPD) that use e-values (like the AVI test), yielding the best of both worlds — detecting when a change happens, with nonparametric, anytime-valid false alarm control.

Strategic Position
E-Detectors are the distribution-free version of the Belief Chart. When an operator does not trust the Gaussian assumption, or when data are heavy-tailed, multi-modal, or contaminated, the E-Detector chart provides the same operational answer — "has the process changed?" — with finite-sample guarantees that hold regardless of the true distribution.

Remark. The Belief Chart gives richer output (posterior run length distribution, regime parameters). The E-Detector gives harder guarantees (provable ARL control). They are complementary. The PBS Health gauge should fuse evidence from both.

2. Mathematical Foundation

2.1 Setup

Observe a stream X₁, X₂, ... sequentially. Before an unknown changepoint ν, data come from a pre-change distribution P. After ν, data come from some post-change distribution Q ≠ P. Both P and Q may be composite (unknown parameters). The goal: raise an alarm as soon as possible after ν, while controlling false alarms when ν = ∞.

2.2 E-Processes

Definition (E-value)
A nonneg random variable E is an e-value for null hypothesis P if 𝔼P[E] ≤ 1. Values E ≫ 1 are evidence against P. By Markov's inequality, P(E ≥ 1/α) ≤ α for any α.
Definition (E-process)
A sequence (Mt)t≥0 is an e-process for P if it is a nonneg supermartingale under P with M₀ ≤ 1. Equivalently: Mτ is an e-value for every stopping time τ. This is the anytime-valid property.

The key construction: given observations X₁, ..., Xt, define a one-step e-value for testing whether Xt came from P:

et = fQ(Xt) / fP(Xt)

When P and Q are simple (known densities), this is the likelihood ratio. The product Mt = ∏s=1t es is the sequential likelihood ratio — an e-process by construction.

2.3 Composite Nulls: The GROW E-value

When P is composite (e.g., "the mean is μ₀ but σ is unknown"), we need e-values valid for all P ∈ 𝒫. The GROW (Growth Rate Optimal in the Worst case) construction:

Definition (GROW E-value for Sub-Gaussian Mean Shift)
For testing H₀: μ = μ₀ against H₁: μ ≠ μ₀ with sub-Gaussian data (bounded in [a, b]):
et(λ) = exp(λ(Xt − μ₀) − ψ(λ))

where ψ(λ) = λ²(b − a)² / 8 is the sub-Gaussian CGF bound. The parameter λ controls the "bet size" — how much evidence we accumulate per observation. Choose λ adaptively via online learning (details in §3).

2.4 From E-Processes to E-Detectors

Definition (E-Detector)
An e-detector is a sequence Nt = Σs=1t Ms:t where Ms:t is an e-process started at time s, evaluated at time t. The alarm is raised at:
τ = inf { t ≥ 1 : Nt ≥ 1/α }
Theorem (ARL Guarantee — Shin, Ramdas & Rinaldo)
For any e-detector with threshold 1/α: 𝔼∞[τ] ≥ 1/α. That is, the average run length under no change is at least 1/α. This holds nonasymptotically, for any data distribution satisfying the null, with no distributional assumptions.

The sum structure is the key insight. Each Ms:t is an e-process that "bets" the changepoint occurred at time s. The sum Nt = Σ Ms:t aggregates evidence across all possible changepoint locations, analogous to the Shiryaev–Roberts statistic but in e-value language.

2.5 CUSUM-Style E-Detector

The sum over all starting times is expensive (O(t) per step). The CUSUM variant uses a running maximum:

Definition (CUSUM E-Detector)
Define the CUSUM e-detector recursively:
NtCUSUM = max(1, Nt−1CUSUM) · et

with N₀ = 1. Alarm when Nt ≥ 1/α. This is O(1) per step and enjoys the same ARL guarantee. The max(1, ·) resets the process when evidence drops below 1, analogous to the classical CUSUM reset.

Theorem (CUSUM E-Detector is an E-Detector)
NtCUSUM ≤ Σs=1t Ms:t for all t. Therefore, the ARL guarantee 𝔼[τ] ≥ 1/α still holds.

3. Algorithm: CUSUM-Style E-Detector for SVEND

3.1 Concrete Implementation for Process Mean Shift

Algorithm (E-Detector for Bounded Data)
Input: Stream X₁, X₂, ...; calibration reference μ₀ (from Phase I or nominal); bounds [a, b] (from spec limits or data range); significance α (default 0.05).
Output: At each t: log-evidence log(Nt), alarm flag, estimated post-change mean.
# Initialize log_N = 0.0 # log of CUSUM e-detector lambda_t = 0.0 # adaptive bet parameter threshold = log(1/α) # e.g., log(20) ≈ 3.0 for α = 0.05 sum_x = 0.0 # running sum for adaptive lambda count = 0 range_sq = (b - a)**2 / 8 # sub-Gaussian variance proxy for each observation X_t: count += 1 # Step 1: Compute one-step log e-value log_e = lambda_t * (X_t - mu_0) - lambda_t**2 * range_sq # Step 2: CUSUM update (reset if below 0 in log space) log_N = max(0, log_N) + log_e # Step 3: Check alarm alarm = (log_N >= threshold) # Step 4: Adapt lambda via ONS (Online Newton Step) # Optimal: lambda* = (X̄ - mu_0) / (2 * range_sq) # Use running mean as proxy for post-change mean sum_x += X_t x_bar = sum_x / count lambda_t = clip((x_bar - mu_0) / (2 * range_sq), -1/(b-a), 1/(b-a)) yield log_N, alarm, x_bar

3.2 Two-Sided Detection

Run two one-sided CUSUM e-detectors in parallel: one for upward shifts (λ > 0) and one for downward shifts (λ < 0). Alarm when either triggers. Combined ARL guarantee: 𝔼[τ] ≥ 1/(2α) by Bonferroni, or use α/2 per side for exact 1/α control.

3.3 Calibration Phase

Critical: Reference Parameter Source
μ₀ must come from Phase I calibration data or the nominal/target value. NEVER compute μ₀ from the full dataset. The bounds [a, b] can be set to [LSL, USL] (spec limits) or estimated from calibration data as [X̄ − 4s, X̄ + 4s]. Using spec limits is preferred — it makes the method truly distribution-free.

Part II
Robust BOCPD via Generalized Bayesian Inference

6. The Outlier Problem in Standard BOCPD

Standard BOCPD (Adams & MacKay, 2007) — which powers the PBS Belief Chart — computes the posterior over run lengths via:

P(rt = ℓ | X1:t) ∝ πℓ(Xt) · P(rt = ℓ | rt−1) · P(rt−1 | X1:t−1)

where πℓ(Xt) is the predictive likelihood of Xt given the data in the current run of length ℓ. The problem:

Failure Mode: Outlier-Induced False Changepoints
A single outlier Xt can have πℓ(Xt) ≈ 0 for all existing runs. This drives the posterior mass toward rt = 0 (new run), triggering a false changepoint. The problem worsens with long runs: the predictive gets sharper (lower variance), making it more sensitive to outliers. This is not a bug in the implementation — it is a fundamental property of standard Bayesian inference using the log-likelihood.

Observed in PBS validation: the bore diameter dataset Phase 3 (overcorrection spike at obs 81–85) triggers a cascade of changepoint detections in standard BOCPD, when the correct interpretation is "transient disturbance, not regime change."

7. Density Power Divergence Scoring

7.1 The Fix: Replace Log-Likelihood with β-Divergence

Altamirano et al. (2023) replace the log-likelihood scoring rule with the density power divergence (DPD), also known as the β-divergence. Instead of:

Standard:  S(x, θ) = log fθ(x)

use:

Robust:  Sβ(x, θ) = −(1/β) fθ(x)β + (1/(1+β)) ∫ fθ(z)1+β dz

where β ∈ (0, 1] is the robustness parameter.

Theorem (Robustness Properties of β-Divergence)
As β → 0, the DPD recovers the log-likelihood (standard Bayesian). As β → 1, the DPD approaches the L² distance between densities. For β > 0, the influence function of the DPD-minimizer is bounded, meaning outliers have bounded effect on parameter estimates. Breakdown point increases with β.

7.2 Conjugate Form for Normal Data

For the Normal-Gamma conjugate family used in PBS, the DPD scoring has a closed-form update. This is the critical result that makes Dm-BOCD practical.

Definition (DPD Score for Normal Distribution)
For X ~ N(μ, σ²), the DPD score at observation x is:
Sβ(x; μ, σ²) = −(1/β)(2πσ²)−β/2 exp(−β(x−μ)²/(2σ²))
+ (1/(1+β))(2πσ²)−β/2 (1+β)−1/2
Proposition (Robustness Intuition)
The exponential term exp(−β(x−μ)²/(2σ²)) acts as a soft truncation. When |x − μ| is large (outlier), this term → 0, meaning the outlier contributes ≈ 0 to the score regardless of how extreme it is. Compare to the standard log-likelihood, where the contribution is −(x−μ)²/(2σ²), which grows without bound.

7.3 Generalized Posterior Update

Replace the standard Bayesian update with a generalized Bayesian update:

πnβ(θ) ∝ πn−1β(θ) · exp(η · Sβ(Xn; θ))

where η > 0 is a learning rate. For the Normal-Gamma conjugate, this yields closed-form sufficient statistic updates with modified counts:

κn = κn−1 + wn
νn = νn−1 + wn
μn = (κn−1μn−1 + wnXn) / κn

where the weight wn replaces the count increment of 1:

wn = exp(−β(Xn − μn−1)² / (2σ²n−1))

Remark. When β = 0, wn = 1 always (standard Bayes). When β > 0, outliers get downweighted automatically. A 5σ outlier with β = 0.3 receives weight w ≈ exp(−0.3 · 25/2) ≈ 0.024. It is almost ignored.

8. Algorithm: Dm-BOCD

Algorithm (Dm-BOCD — Divergence-Minimizing BOCPD)
Input: Stream X₁, X₂, ...; prior hyperparameters (μ₀, κ₀, α₀, β₀); changepoint prior hazard h; robustness parameter β ∈ [0.1, 0.5].
Output: At each t: posterior P(rt | X1:t), MAP run length, regime parameters.
# Initialize: same structure as standard BOCPD # For each active run length ℓ, maintain: # (kappa_l, mu_l, alpha_l, beta_l) — sufficient stats for each observation X_t: # Step 1: Compute predictive score for each run length for each active run length ℓ: mu_pred = mu_l sigma2_pred = beta_l * (kappa_l + 1) / (alpha_l * kappa_l) # DPD predictive (Student-t with robust scoring) # Compute weight for this observation z = (X_t - mu_pred)**2 / sigma2_pred w = exp(-beta_param * z / 2) # DPD predictive log-score (replaces log predictive) log_pred_score[ℓ] = dpd_predictive_log_score( X_t, mu_pred, sigma2_pred, beta_param) # Step 2: Growth probabilities (identical to standard BOCPD) log_growth[ℓ+1] = log_joint[ℓ] + log_pred_score[ℓ] + log(1-h) # Step 3: Changepoint probability log_cp = logsumexp(log_joint[:] + log_pred_score[:]) + log(h) # Step 4: Update sufficient statistics with ROBUST weights for each active run length ℓ: w = exp(-beta_param * (X_t - mu_l)**2 / (2 * sigma2_l)) kappa_l_new = kappa_l + w mu_l_new = (kappa_l * mu_l + w * X_t) / kappa_l_new alpha_l_new = alpha_l + w / 2 beta_l_new = beta_l + w * kappa_l * (X_t - mu_l)**2 / (2 * kappa_l_new) # Step 5: Normalize, truncate (same as standard) # Truncate to top K=200 run lengths

8.1 Choosing β

β ValueBehaviorWhen to Use
0.0Standard BOCPD (no robustness)Clean data, validated Normal assumption
0.1Mild robustness, close to standardDefault for most manufacturing processes
0.3Moderate robustnessProcesses with known measurement outliers, gage issues
0.5Strong robustnessHeavy-tailed data, contaminated streams
> 0.5Very conservative, may miss real shiftsRarely recommended
Anti-Pattern: Setting β Too High
High β makes BOCPD robust to outliers but also slower to detect real shifts. The tradeoff is fundamental: robustness = reduced sensitivity. Default β = 0.1 for SVEND. Surface β as a user-adjustable parameter labeled "Outlier Robustness" with a slider.

9. Integration with PBS Belief Chart

9.1 Drop-In Replacement

Dm-BOCD has the identical interface as standard BOCPD. Same inputs, same outputs. The only change is the internal scoring rule and sufficient statistic update. This means:

New output: for each observation, report the weight wt assigned by the DPD. Observations with w < 0.5 are being downweighted as potential outliers. This provides an automatic outlier flagging mechanism as a side-effect of the robust inference.

Part III
Conformal Multivariate Process Monitoring

10. From Univariate Charts to Multivariate Anomaly Detection

SVEND's existing Conformal Control Chart (from Burger §3) monitors a single quality characteristic. But real processes have 5–50 correlated variables (temperatures, pressures, flow rates, chemical concentrations). Monitoring each with a separate chart is statistically unsound (multiplicity) and operationally overwhelming.

Burger's §4 proposes the solution: reframe multivariate monitoring as anomaly detection, wrap any anomaly detector in conformal calibration, and monitor a single p-value chart.

Key Insight
Classical multivariate SPC (Hotelling T²) assumes multivariate normality and known covariance. The conformal approach: train any anomaly detection model on in-control data (autoencoder, isolation forest, one-class SVM, etc.), then use conformal calibration to convert its raw scores into a p-value with guaranteed false alarm rate α, regardless of whether the model is correct or the data is Normal.

11. Conformal Calibration of Anomaly Scores

11.1 The Calibration Procedure

Definition (Nonconformity Score)
Let A(·) be any anomaly detection function that maps a multivariate observation x ∈ ℝd to a scalar score s ∈ ℝ, where higher scores indicate greater anomalousness. Examples: reconstruction error of an autoencoder, isolation depth, Mahalanobis distance, negative log-likelihood of a density estimator.
Definition (Calibration Set)
From Phase I in-control data, compute anomaly scores for each calibration observation: Scal = {A(x1), A(x2), ..., A(xm)}. This is the empirical distribution of "normal" anomaly scores.
Definition (Conformal p-Value)
For a new observation xt with score st = A(xt), the conformal p-value is:
pt = (1 + |{i : Scal,i ≥ st}|) / (m + 1)

This is the rank of st among the calibration scores, normalized. Small p-value = observation is more anomalous than most calibration data.

Theorem (Finite-Sample Validity — Vovk et al., 2005)
If xt is exchangeable with the calibration data (i.e., the process is in control), then P(pt ≤ α) ≤ α for any α. This holds exactly in finite samples, for any distribution, for any anomaly scoring function A(·).

It requires no distributional assumptions, handles mixed variable types, and the conformal wrapper guarantees valid p-values regardless of model quality. Add Mahalanobis as a "classical" fallback. Autoencoder as a premium option for complex processes.

12.1 Algorithm

class ConformalMultivariateMonitor: def __init__(self, calibration_data, alpha=0.05, method='isolation_forest'): # Step 1: Fit anomaly detector on calibration data self.detector = fit_anomaly_detector(calibration_data, method) # Step 2: Compute calibration scores self.cal_scores = sorted([ self.detector.score(x) for x in calibration_data ]) self.m = len(self.cal_scores) self.alpha = alpha def monitor(self, x_new): # Step 3: Score new observation s_new = self.detector.score(x_new) # Step 4: Compute conformal p-value via rank # Number of calibration scores ≥ s_new rank = bisect_left(self.cal_scores, s_new) n_geq = self.m - rank p_value = (1 + n_geq) / (self.m + 1) # Step 5: Signal anomaly = (p_value <= self.alpha) return { 'p_value': p_value, 'anomaly_score': s_new, 'anomaly': anomaly, 'evidence_strength': classify_p(p_value) } def classify_p(p): if p > 0.10: return 'NORMAL' if p > 0.05: return 'WARNING' if p > 0.01: return 'ANOMALY' return 'SEVERE ANOMALY'

14. Anti-Patterns (All Methods)

Anti-Pattern 1: Computing Reference from Full Data
Applies to: E-Detectors, Conformal Multivariate.
The reference μ₀ (E-Detector) and calibration scores (Conformal) must come from Phase I / calibration data only. Using the full dataset makes shifts "expected" under the null, destroying detection power. This is the same bug fixed in PBS E-value computation.
Anti-Pattern 2: Conformal Calibration Set Too Small
Applies to: Conformal Multivariate.
The conformal p-value has resolution 1/(m+1). With m = 20 calibration points, the smallest achievable p-value is 1/21 ≈ 0.048 — barely below α = 0.05. Need m ≥ 100 for useful resolution. m ≥ 200 recommended. Flag if calibration set is too small.
Anti-Pattern 3: Using β-Divergence Without Understanding Tradeoff
Applies to: Robust BOCPD.
β > 0 is not free: robustness costs detection speed. A real 1σ shift may take 2–3× more observations to detect at β = 0.3 vs β = 0. Surface the tradeoff in the UI. Never silently set β = 0.5.
Anti-Pattern 4: E-Detector Bounds Too Wide
Applies to: E-Detectors.
The sub-Gaussian bound (b − a)²/8 appears in the exponent. If [a, b] = [LSL, USL] and spec range is much wider than actual process variation, the bet size λ will be tiny, evidence accumulates slowly. Use tighter bounds from calibration data when possible: [X̄ − 4s, X̄ + 4s].
Anti-Pattern 5: Anomaly Detector Trained on Contaminated Data
Applies to: Conformal Multivariate.
If calibration data contains outliers or out-of-control periods, the anomaly detector learns those as "normal." The conformal guarantee still holds (p-values are valid) but detection power drops because the calibration distribution is polluted. Pre-screen calibration data with univariate PBS or standard control charts.
Anti-Pattern 6: Displaying Raw p-Values Without Context
Applies to: Conformal Multivariate.
A p-value of 0.03 is not "3% defective" — it means "this observation is more anomalous than 97% of calibration data." Always show the natural-language interpretation. Use −log₁₀(p) scale for the chart. Never expose raw p-values to operators without explanation.
Anti-Pattern 7: Running E-Detector and BOCPD Independently Without Fusion
Applies to: All methods in PBS context.
When both E-Detector and BOCPD are available, the PBS Health gauge should fuse both signals. Do not show contradictory alarms without reconciliation. Priority: if E-Detector alarms and BOCPD does not, this likely means BOCPD's Normal assumption is wrong. If BOCPD alarms and E-Detector does not, BOCPD may be reacting to an outlier (use Robust BOCPD).
