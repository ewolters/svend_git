## CRITICAL Findings

### CRIT-01. Race condition — `UsageLog.log_request` loses usage data
- **Location:** `chat/models.py:123-146`
- **Issue:** `get_or_create` then `self.field += N; self.save()`. Two concurrent requests read same row, both increment in-memory, one write overwrites the other. Breaks rate limiting accuracy.
- **Fix:** Use `F()` expressions for atomic increment (pattern already exists in `User.increment_queries()`).

### CRIT-02. Race condition — `UserQuota.add_file/remove_file` drifts from reality
- **Location:** `files/models.py:216-226`
- **Issue:** `used_bytes` and `file_count` read in Python, arithmetic done in-memory, then `save()`. Concurrent uploads can exceed storage quotas.
- **Fix:** `UserQuota.objects.filter(pk=self.pk).update(used_bytes=F('used_bytes') + size_bytes, file_count=F('file_count') + 1)`

### CRIT-03. Race condition — `Board.save()` version increment non-atomic
- **Location:** `agents_api/models.py:420-426`
- **Issue:** `super().save()` then separate `F('version') + 1` update. Concurrent saves between the two operations get same version number, defeating conflict detection.
- **Fix:** Atomic version increment via `select_for_update()` or single-query update.

### CRIT-04. All `/app/*` TemplateViews served without authentication
- **Location:** `svend/urls.py:174-331`
- **Issue:** ~40 routes (dashboard, settings, FMEA, hoshin, VSM, ISO, safety, A3, etc.) use `TemplateView.as_view()` with NO `login_required`. Anonymous users see the full application shell.
- **Fix:** Wrap with `login_required` or add middleware checking `request.path.startswith('/app/')`.

### CRIT-05. Shared conversation exposes reasoning traces and tool calls
- **Location:** `chat/views.py:9-36`
- **Issue:** `shared_conversation` requires NO auth, returns full conversation including `reasoning_trace` and `tool_calls`. No expiry. Anyone with the share UUID reads internal diagnostics.
- **Fix:** Remove `reasoning_trace`/`tool_calls` from public response. Add `expires_at` check.

## HIGH Findings

### HIGH-01. Webhook dispatch has no tenant isolation
- **Location:** `notifications/webhook_delivery.py:40`
- **Issue:** `WebhookEndpoint.objects.filter(is_active=True)` — no tenant filter. Tenant A's events dispatched to Tenant B's wildcard endpoints.
- **Impact:** SOC 2 CC6.3 violation. Cross-tenant data breach.
- **Fix:** Add `tenant_id` parameter to `dispatch_event()`, filter endpoints by tenant.

### HIGH-02. Safety card-to-FMEA cross-tenant injection
- **Location:** `safety/views.py:443-444`
- **Issue:** Card is tenant-filtered but FMEA lookup uses `get_object_or_404(FMEA, id=fmea_id)` with no tenant check. User can inject findings into another org's FMEA.
- **Fix:** `get_object_or_404(FMEA, id=fmea_id, owner__memberships__tenant=tenant)`

### HIGH-03. Tenant isolation gaps in Workbench, KnowledgeGraph, EpistemicLog
- **Location:** `workbench/models.py`
- **Issue:** Only `user` FK, no `tenant` FK. A single missed filter in any view leaks cross-org data.
- **Fix:** Add tenant FK + model-level manager filtering.

### HIGH-04. Tenant isolation gaps in agents_api models
- **Location:** `agents_api/models.py`
- **Issue:** `Report`, `IshikawaDiagram`, `CEMatrix`, `ValueStreamMap`, `PlantSimulation`, `ManagementReview`, `SupplierRecord` — all have `user` FK but no `tenant` FK.
- **Fix:** Add tenant FK or comprehensive view-level isolation with tests.

### HIGH-05. `UserQuota.get_or_create_for_user` tier mapping never matches
- **Location:** `files/models.py:248-272`
- **Issue:** Tier keys use lowercase (`"free"`, `"beta"`, `"pro"`) but `Tier` enum is uppercase (`"FREE"`, `"FOUNDER"`, `"PRO"`). ALL users fall through to free-tier quota defaults.
- **Fix:** Update keys to match `Tier` enum values. Add TEAM/ENTERPRISE entries.

### HIGH-06. `check_rate_limit` reads non-existent `user.subscription_tier`
- **Location:** `agents_api/models.py:311`
- **Issue:** `getattr(user, "subscription_tier", "FREE")` — User model has `tier`, not `subscription_tier`. All paid users get free-tier rate limits.
- **Fix:** Change to `getattr(user, "tier", "FREE")`.

### HIGH-07. Forge `APIKey` — duplicate model, no brute-force protection
- **Location:** `forge/models.py:34-54`, `forge/views.py:33-74`
- **Issue:** Separate from `accounts.APIKey`. SHA-256 hash lookup with no rate limiting on failures. No expiry support.
- **Fix:** Consolidate with `accounts.APIKey` or add rate limiting + expiry.

### HIGH-08. `ProcessEdge` allows duplicate edges (missing unique constraint)
- **Location:** `graph/models.py:210-352`
- **Issue:** No `unique_together` on `(graph, source, target, relation_type)`. Duplicate edges corrupt Bayesian posterior calculations.
- **Fix:** Add `UniqueConstraint(fields=["graph", "source", "target", "relation_type"])`.

### HIGH-09. Widespread `@csrf_exempt` on session-accessible state-changing endpoints
- **Location:** `safety/views.py`, `loop/views.py`, `notifications/views.py`, `accounts/api_key_views.py`
- **Issue:** ~30+ endpoints. While JSON API endpoints are partially protected, these also accept session cookies.
- **Fix:** Remove `@csrf_exempt` or require API key-only auth (no session).

### HIGH-10. `is_public` file flag settable without validation
- **Location:** `files/views.py:234-244`
- **Issue:** Generic PATCH accepts `is_public` with no tier/permission check. Bypasses share-link flow.
- **Fix:** Only allow `is_public=True` through the dedicated `create_share_link` endpoint.

### HIGH-11. `generate_room_code` uses non-crypto `random`
- **Location:** `agents_api/models.py:363-368`
- **Issue:** `random.choices()` for room codes, no retry logic on unique constraint failure.
- **Fix:** Use `secrets.token_hex(3)` + retry loop.

### HIGH-12. `EdgeEvidence` CASCADE contradicts immutability guarantee
- **Location:** `graph/models.py:385-389`
- **Issue:** Docstring says "Evidence is never deleted (GRAPH-001 section 16.3)" but `on_delete=CASCADE` destroys evidence when edges are deleted.
- **Fix:** Change to `on_delete=PROTECT`, require edge soft-deletion.

### HIGH-13. Zero tests for entire `loop` app — 4,262 lines of core QMS logic
- **Location:** `loop/` (models.py, views.py, services.py, readiness.py, evaluator.py, bayesian.py)
- **Issue:** Signals, Commitments, Mode Transitions, Process Confirmations, Forced Failure Tests, FMIS, Auditor Portal (external auth tokens) — all completely untested.
- **Impact:** The core QMS loop mechanism has no regression protection whatsoever.

## MEDIUM Findings

### MED-01. CAPA field changes logged against stale values during status transition
- **Location:** `agents_api/capa_views.py:230-290`
- **Issue:** `_log_field_changes()` called after fields already set — `old_val` reads the new value. Audit trail is silently incomplete.
- **Fix:** Call `_log_field_changes()` before the first `setattr` loop, or read old values from DB.

### MED-02. CAPA DELETE permanently destroys quality records
- **Location:** `agents_api/capa_views.py:223-224`
- **Issue:** Hard `delete()` on regulated quality records. `CAPAStatusChange` records cascade-deleted.
- **Fix:** Soft-delete with `is_deleted` flag, or remove DELETE endpoint.

### MED-03. Graph posterior computation heuristic produces misleading values
- **Location:** `graph/service.py:287-329`
- **Issue:** Arbitrary contribution mapping (effect_size 5.0 = maximal confidence, same as 50.0). p_value interaction with effect_size is unintuitive.
- **Fix:** Document limitations in UI, consider proper Bayesian update.

### MED-04. Readiness score hardcodes "good" recurrence rate (+8 points)

### MED-05. Tenant isolation disabled globally

### MED-06. CSP allows `'unsafe-eval'`

### MED-07. CSP drift between Caddy and Django

### MED-08. No per-user LLM cost cap
- **Location:** `inference/flywheel.py`, `agents_api/*_views.py` (15+ call sites)
- **Issue:** Various `max_tokens` (200-4096) but no aggregate daily spend limit per user.
- **Fix:** Track cumulative tokens in DB, reject over budget.

### MED-09. `ModelVersion.activate()` TOCTOU race
- **Location:** `chat/models.py:61-74`
- **Issue:** Deactivate-then-activate without `select_for_update`. Concurrent activations could leave no active model.
- **Fix:** Wrap in `transaction.atomic()` with `select_for_update()`.

### MED-10. `WebhookEndpoint.secret` stored in plaintext
- **Location:** `notifications/models.py:175`
- **Issue:** Used for HMAC signing but stored unencrypted. Unlike API keys which use `hash_token()`.
- **Fix:** Use `EncryptedCharField` from `core.encryption`.

### MED-11. `Forge.Job` allows both `api_key` and `user` to be null
- **Location:** `forge/models.py:62-70`
- **Issue:** Job can exist with no owner at all.

### MED-12. `DSWResult` and `TriageResult` use CharField PK instead of UUID
- **Location:** `agents_api/models.py:41, 90`
- **Issue:** User-controlled PKs, violates DAT-001 convention, injection/collision risk.
- **Fix:** Migrate to UUIDField.

### MED-13. `Workflow.steps` uses TextField instead of JSONField
- **Location:** `agents_api/models.py:27`
- **Issue:** No database-level JSON validation. Same for `SavedModel.metrics` and `feature_names`.
- **Fix:** Migrate to `JSONField(default=list)`.

### MED-14. Graph edge: no validation source/target in same graph
- **Location:** `graph/models.py:244-255`
- **Issue:** Edge could link non-shared nodes from different graphs.
- **Fix:** Add `clean()` validation.

### MED-15. `KnowledgeGraph` JSON-based graph mutations not concurrency-safe
- **Location:** `workbench/models.py:535-759`
- **Issue:** Full JSON read-mutate-write cycle. Concurrent edits overwrite each other.
- **Fix:** Add version field for optimistic locking.

### MED-16. `Notification.save()` immutability check doubles write cost
- **Location:** `notifications/models.py:102-113`
- **Issue:** Every save fetches current DB row to check immutable fields.
- **Fix:** Use `pre_save` signal with `update_fields` checking.

### MED-17. `LoginAttempt` records accumulate indefinitely
- **Location:** `accounts/models.py:145-195`
- **Issue:** No cleanup mechanism. Table grows unboundedly.
- **Fix:** Add periodic purge task.

### MED-18. Error messages in agent views expose exception details
- **Location:** `agents_api/views.py:68`
- **Issue:** `str(e)[:200]` returned to client. Internal paths/state could leak.

### MED-19. `site_duration` and `funnel_event` endpoints lack rate limiting
- **Location:** `api/views.py`
- **Issue:** `AllowAny` POST with no throttling. DB flood possible.
- **Fix:** Add `AnonRateThrottle`.

### MED-20. Email tracking uses predictable recipient UUIDs
- **Location:** `api/views.py:1382-1432`
- **Issue:** No signature verification. Anyone with a UUID can mark emails opened/clicked.

### MED-21. Auditor portal token values exposed in list API
- **Location:** `loop/views.py:2118-2138`
- **Issue:** GET returns full token value. Token IS the credential for unauthenticated portal access.

### MED-22. `agents_api/tests.py` — 0% effective coverage (skipTest abuse)

- **Issue:** 7 `skipTest()` calls as flow control. Tests appear green but exercise nothing. Violates TST-001 section 10.6.
